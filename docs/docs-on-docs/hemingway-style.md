# Hemingway Style

All writing — docs, comments, commit messages, PR descriptions — follows these rules:

**Short sentences.** Prefer under 25 words.

**Active voice.** "Converts queries" not "Queries are converted."

**Simple words.** "Use" not "utilize." "Start" not "initiate."

**Direct statements.** No hedging. Drop "might", "could", "possibly."

**Concrete details.** Specific beats abstract. Show the thing.

**No fluff.** No marketing speak, no filler, no preamble.

**Show real usage patterns.** Examples beat descriptions. Keep examples short and illustrative. Don't duplicate actual code from the codebase.
