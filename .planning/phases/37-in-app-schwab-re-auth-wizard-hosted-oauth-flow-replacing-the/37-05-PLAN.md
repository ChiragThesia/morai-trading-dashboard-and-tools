<objective>
Add the server-side proxy: two JWT-gated HTTP routes that forward the wizard's start/exchange
calls to the sidecar admin endpoints via the 37-02 adapter, mapping Results to generic HTTP
responses that never leak sidecar detail.

Purpose: the sidecar has no public Railway domain (GW-05), so the server is the only thing that
can reach it; and the server already owns the Supabase JWT boundary, so any authed user is the
operator (no new role system). This plan is pure plumbing over established patterns
(settings.routes.ts), plus one deliberate security decision: the re-auth surface is HTTP-only —
no MCP tool is added, because a tool that mints Schwab authorize URLs is a privileged surface we
do not expose to the MCP client. The codebase's usual HTTP+MCP pairing is explicitly overridden
here.
Output: config.ts (+SIDECAR_ADMIN_TOKEN), reauth.routes.ts + test, main.ts wiring.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → server /api/reauth/* | The only public entry to the re-auth flow; gated by the existing Supabase JWT (operator-only). |
| server → sidecar | The server is the sole caller of the private sidecar admin endpoints; it must not surface sidecar error detail back to the browser. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-37-04 | Elevation of Privilege | /api/reauth/* mount | high | mitigate | Routes mounted inside the existing authReadGroup (Supabase JWT); no unauthenticated surface; NO MCP tool exposes this capability (privileged auth-minting surface kept off the MCP client). |
| T-37-06 | Information Disclosure | reauth.routes.ts error mapping | medium | mitigate | `!result.ok` → generic `{error:"internal"}` 500; success re-parsed through the strict contract; sidecar detail / code / redirect URL never reach the browser (leak-free assertion in the route test). |
| T-37-02 | Information Disclosure | server → sidecar adapter | high | mitigate | Adapter (37-02) logs only constructor.name; the route never logs the request body or use-case error message. |
</threat_model>
