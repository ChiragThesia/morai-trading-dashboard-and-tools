## Spacing Scale

Declared values (must be multiples of 4), matching the project's existing convention:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, step-chip internal padding |
| sm | 8px | Gap between step copy and CTA button; banner row gap |
| md | 16px | Dialog content padding (`DialogContent p-4`); banner horizontal padding (existing, unchanged) |
| lg | 24px | Gap between step-indicator row and step body |
| xl | 32px | Not used this phase |

Exceptions:

- **44px (`min-h-11`) touch target** on every clickable control in the wizard — "Reconnect" (banner), "Authorize with Schwab", "Retry", "Close". Use `<Button size="touch">` verbatim (already renders `min-h-11` below `lg:` and collapses to the desktop `xs` size at `lg:` per the existing Phase-35 responsive contract). No new touch-target primitive needed.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (banner, both red + amber states) | **Reconnect** |
| Primary CTA (wizard, per step) | **Authorize with Schwab** (opens the Schwab authorize URL via a full same-tab redirect) |
| Step indicator | **Trader (1/2)** → **Market (2/2)** — plain-text uppercase labels in step chips, never numbered as "Step 1 of 2" prose |
| Step idle/waiting copy | "Click Authorize with Schwab to reconnect the {trader/market} app." |
| Step in-flight copy (post-redirect, exchanging) | "Confirming…" (text-dim, matches `RuleSettingsModal`'s "Previewing…" idiom — no spinner component needed, plain text is the project's existing convention) |
| Per-app success | "{Trader/Market} connected." (text-up, with a lucide `Check` icon) — auto-advances to the next step; on the final app, replaces the wizard body with the Done state below |
| Per-app failure (inline, scoped to just that step) | "{Trader/Market} reconnect failed — Schwab didn't confirm a fresh token." + **Retry** button that re-runs only this app's authorize step. The other app's already-succeeded state is untouched and stays visible as "{App} connected." |
| Done state (both apps connected) | "Reconnected. Live data resumes on the next status check." + **Close** button (`variant="secondary"`) |
| Destructive confirmation | None in this phase — there is no delete/destroy action. Retry is idempotent (re-opens the Schwab authorize URL for the failed app only) and needs no confirmation dialog. |

**Content constraint (locked, non-negotiable):** the authorization `code` query param and the full callback redirect URL must never render in any copy, toast, tooltip, or error string, anywhere in this contract. Error copy above is deliberately generic ("didn't confirm a fresh token") — it never echoes server error detail that could leak the code or state nonce.

**Banner copy change note:** the existing `AuthExpiredBanner` copy ("Run `auth setup` to reconnect...") is superseded as the *primary* instruction now that a button exists — replace it with the CTA-first copy below. The CLI fallback stays documented in `docs/operations/schwab-reauth-runbook.md`, not repeated in the banner text.

| Banner state | New copy |
|---|---|
| Red (`trader.status === "AUTH_EXPIRED"`) | "Schwab auth expired. Live data may be stale." + **Reconnect** button |
| Amber — market expired only | "Schwab market app auth expired — chain data fell back to CBOE." + **Reconnect** button |
| Amber — T-24h warning | "Schwab auth expires soon. Reconnect within 24 hours to avoid an outage." + **Reconnect** button |

## Interaction Contract (wizard states)

Container: `Dialog` opened by the banner's **Reconnect** button (`DialogTrigger`). `DialogContent` default sizing (`w-full max-w-[calc(100%-2rem)] sm:max-w-sm`) is sufficient — this is copy + one CTA per step, not a data-dense form; do not widen it to `RuleSettingsModal`'s `sm:max-w-lg`.

Step-chip row (non-interactive, purely a progress indicator — not clickable, the flow is not user-navigable backward):

- Render two chips styled with `buttonClass({ variant: "toggle", tone: "violet", active, size: "xs" })` on a plain `<span>`/`<div>`, never a real `<button>` (they don't do anything on click).
- `active=true` (filled violet) once that app's step is the current one OR already succeeded.
- A succeeded app's chip stays filled/violet permanently (do not revert to inactive), even while the other app's step is still in progress or retrying — this is what makes partial-failure isolation visible: user always sees which app is done.

State machine per app (trader, then market — sequential entry, but failure isolation per CONTEXT.md):

1. **idle** → idle copy + "Authorize with Schwab" button (`variant="primary" tone="violet" size="touch"`)
2. click → browser navigates away (full redirect, same tab) — no in-app loading state needed here, the tab itself unloads
3. **return** (app reloads at `https://morai.wtf?code=&state=`) → wizard auto-resumes at the step matching the app the state nonce belongs to, strips `code`/`state` via `history.replaceState` before any render, shows "Confirming…"
4. **success** → "{App} connected." (text-up + Check icon) → auto-advance: trader success moves to the Market idle step; market success moves to the Done state
5. **failure** → inline error copy (text-down) + scoped **Retry** button (`variant="secondary" size="touch"`) that re-enters state 1 for that app only — the other app's success/idle state is unaffected

Done state replaces the step body entirely (both chips filled violet) — see Copywriting Contract.
