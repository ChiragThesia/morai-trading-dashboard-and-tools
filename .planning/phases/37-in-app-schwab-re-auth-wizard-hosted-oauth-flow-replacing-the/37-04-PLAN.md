<objective>
Add the two sidecar admin endpoints that are the heart of the wizard: /start (mint authorize URL
+ nonce) and /exchange (consume nonce, exchange the redirect for tokens, anchor refresh, re-init
the session), both gated by the SIDECAR_ADMIN_TOKEN shared secret.

Purpose: this is where the OAuth mechanics proven live in seed_token.py become admin routes.
Everything hard was de-risked in 37-01 (nonce table) and 37-03 (anchoring writer + reinit); this
plan wires them into two FastAPI handlers with the CSRF nonce, the per-app freshness success gate,
partial-failure isolation, and the log-nothing-sensitive discipline.
Output: reauth_admin.py (two endpoints + header guard + nonce persistence), router registered in
main.py, test_reauth_admin.py, reauth_nonces test fixture in conftest.py.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server → sidecar admin endpoints | Untrusted-until-authenticated calls that MINT and EXCHANGE Schwab tokens — gated by the shared secret; the sidecar has no other auth. |
| browser-originated redirect URL → sidecar | The `state`/`code` in the redirect are attacker-influenceable; the nonce table + Schwab's own code validation are the controls. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-37-03 | Elevation of Privilege | admin-token guard (reauth_admin.py) | critical | mitigate | Both endpoints require X-Sidecar-Admin-Token; compared with `hmac.compare_digest` (constant-time); sidecar stays private-net only (GW-05). |
| T-37-01 | Spoofing/Tampering | nonce consume (/exchange) | high | mitigate | Atomic `DELETE ... RETURNING app_id` with a 10-minute TTL predicate — single-use, replay-killed; an unknown/expired/consumed state is rejected without guessing an app_id. |
| T-37-02 | Information Disclosure | exchange failure path | high | mitigate | Log only `type(exc).__name__`; never str(exc), the redirect URL, or the code (RESEARCH Pitfall 8). |
| T-37-05 | Tampering | reinit call after exchange | high | mitigate | Calls 37-03's reinit_schwab_session (cancel+await old tasks before recreate; lock held throughout). |
| T-37-08 | Tampering | stale/reused authorization code | low | accept | Schwab's own ~30s code TTL is the backstop; the exchange makes no extra await-able hop before calling Schwab (RESEARCH Pitfall 6). |
</threat_model>
