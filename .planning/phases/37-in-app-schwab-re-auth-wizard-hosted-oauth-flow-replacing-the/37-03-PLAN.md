<objective>
Add the sidecar's in-process re-init plumbing that the wizard exchange (37-04) calls after a
successful token write: config fields for the two new secrets, a refresh_issued_at-anchoring
token writer, and a reinit_schwab_session helper that bounces the Schwab clients + background
tasks while holding the advisory lock.

Purpose: three of this phase's non-plumbing traps live here (RESEARCH). The routine token writer
never anchors refresh_issued_at, so without a dedicated writer the wizard's freshness gate can
never pass; the streamer/keepalive tasks each capture the client in a local variable once, so
merely rebuilding app.state.trader_client does not reach them; and re-init must never release the
GW-04 advisory lock or start a second streamer session. This plan closes all three, independently
of the endpoints that call them.
Output: config.py (+2 fields), token_store.make_reauth_writer, main.py (app.state.cfg + tasks on
app.state + reinit_schwab_session) — all tested.
</objective>

|-----------|----------|-----------|----------|-------------|-----------------|
| T-37-05 | Tampering | reinit_schwab_session (main.py) | high | mitigate | Old keepalive/streamer tasks are cancelled AND awaited BEFORE the new ones are created — never two live streamer sessions; the advisory lock is held continuously (reinit never closes lock_conn or mutates has_lock); reinit no-ops when has_lock is False (never rebuilds clients this instance has no right to hold). |
