<objective>
Give the exit advisor a runtime-override seam for its TAKE (+15%/+10%/+5%) and STOP (−50%/−25%)
rungs, each with an arm/disarm hysteresis pair (CONTEXT.md "Exit advisor"). Follow the
optional-param idiom: `evaluateExit`'s current 3-arg signature gains an optional 4th `config`
argument defaulting to the `exit-rules.ts` constants, so omitting it reproduces today's verdicts
byte-identically. Add the pure `resolveExitRuleConfig(overrides?)` merge function in its own
exits-context `domain/rule-config.ts` (never a cross-context settings module — architecture
rule 7).

Purpose: unblock the exits worker-wiring (29-11) which reads overrides fresh each
compute-exit-advice run and passes the resolved config as the 4th arg.
Output: `resolveExitRuleConfig` + a config-threaded `evaluateExit`.

Note (RESEARCH Pitfall 2): the hard hysteresis VALIDATION (disarm ordered vs arm, pair
completeness) lives at the contract boundary (29-02). `resolveExitRuleConfig` performs the pure
merge over validated input; it defaults each rung's label + arm + disarm from `TAKE_RUNGS` /
`STOP_RUNGS`.
</objective>
