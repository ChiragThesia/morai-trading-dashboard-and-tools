<objective>
Wire the two cascading picker knobs — the VIX ladder tier boundaries (0/15/20/25) and the sizing
tier contract counts ({low:2, normal:2, elevated:1, crisis:0}) — behind optional parameters that
default to today's constants. The VIX ladder is read by three consumers (resolveEntryGate,
sizing's SIZING_TIERS build, and candidate-selection's autoTuneTargetDelta via vixLadderFloor),
so this plan introduces ONE pure ladder-rebuild helper the merge fn and every consumer share.

Open-question resolution (orchestrator, RESEARCH Open Question 1): overriding the ladder
boundaries does NOT shift the gate's penalty-floor / block-arm hysteresis constants
(`VIX_PENALTY_FLOOR`, `VIX_BLOCK_ARM`, `GATE_PENALTY_FLOOR_MULTIPLIER`, etc.) — those are
excluded/code-only per CONTEXT.md. Ladder tiers move; gate arm/disarm points stay put.

Purpose: unblock the picker merge fn (29-07) and worker wiring (29-10).
Output: `resolveVixLadder(override?)` + an optional `vixLadder` param on `resolveEntryGate`;
`resolveSizingTier(vix, override?)` rebuild path + an exported `DEFAULT_TIER_CONTRACTS`.
NOT in scope: candidate-selection's autoTuneTargetDelta ladder param (that file is owned by 29-03;
its autoTune already reads VIX_LADDER — the worker wiring in 29-10 will feed it the resolved
ladder-derived floor, no signature change needed there for this plan).
</objective>
