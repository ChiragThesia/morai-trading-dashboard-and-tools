<objective>
Build the gear-icon settings modal (CONTEXT.md UI lock). Add a gear icon to the top-right of the
existing `Shell.tsx` top bar (the `NAV_TABS` bar with Overview/Analyzer/Journal), opening a
`Dialog` modal grouped by engine: Entry/Picker · Exit Advisor · Regime Bands. Each knob shows its
effective value and, when overridden, its default alongside (so the user sees drift from
baseline). Each group has a reset-to-defaults button that clears that group. Use the shared
`<Button>` primitive (Phase 21) per the CONTEXT.md lock, and the existing `Dialog` wrapper — do
not hand-roll a modal.

`useRuleSettings` mirrors `useRuleTags.ts` exactly: a `useQuery` for GET, a non-optimistic
`useMutation` for PUT (never flips local state before the PUT resolves), `invalidateQueries` on
success, per-group error mapping. The reset button PUTs `{ <group>: null }`.

This plan has a blocking human-verify checkpoint for the visual placement/grouping (per
29-VALIDATION.md Manual-Only). `human_verify_mode` is not end-of-phase here; use a checkpoint.

Purpose: the operator can read + edit the curated knobs from the dashboard.
Output: the hook, the modal, the gear trigger.
</objective>
