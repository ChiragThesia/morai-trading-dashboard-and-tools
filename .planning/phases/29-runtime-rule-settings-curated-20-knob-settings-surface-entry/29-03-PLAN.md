<objective>
Add the missing runtime-override parameter seams to three picker-domain files, following the
existing `weights?`/`effectiveDeltaMin?` optional-param idiom (Pattern 1, RESEARCH.md). Every
new parameter is optional and defaults to the SAME module constant it replaces, so omitting it
reproduces today's output byte-identically — the invariant the Phase 27 backtest leakage oracle
(BT-02) depends on.

Scope (curated knobs, CONTEXT.md "Entry/picker"):
- `candidate-selection.ts`: `DELTA_BAND_MAX` (read at the delta filter), `FRONT_DTE_MIN` /
  `FRONT_DTE_MAX` (read at the front-DTE filter) — currently hard-coded module constants with no
  seam. (`effectiveDeltaMin`, `backDteMinGap`, `backDteMaxGap` seams already exist — leave them.)
- `rules.ts`: `debitFitFraction(debit)` reads `DEBIT_IDEAL_MIN`/`DEBIT_IDEAL_MAX` directly.
- `brakes.ts`: `maxOpenTripped(openCount)` reads `MAX_OPEN_CALENDARS` directly.

Purpose: unblock the picker merge fn (29-07) and the worker wiring (29-10) which pass resolved
override values into these seams.
Output: three functions gain optional, default-preserving parameters + omission-regression tests.
NOT in scope: the excluded/code-only knobs (normalizers, cheap-floor/expensive-zero debit edges,
cooldown, fill haircut) — leave them as bare constants.
</objective>
