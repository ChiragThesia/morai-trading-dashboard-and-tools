<objective>
Reflow the Overview screen body for mobile: (1) split the 10-chip `PillHeader` into a single-line
priority row (SPX / net γ/1% / VIX / book) plus a scroll-snap `ChipRail` of the other 6 metrics below
`lg`, folding back to today's one flex-wrap row at `lg` (CONTEXT item 1, D-06); (2) reorder the
3-column grid via CSS `order` so the payoff hero renders above the now-collapsed `MarketRail` on phones
(CONTEXT item 2, D-03); (3) de-sticky the `PillHeader` below `lg` so Shell's header is the only sticky
layer; and (4) make the payoff chart full-bleed and hide the redundant "view-only · Analyzer →" chrome
below `lg` (D-10). Positions stay a `<table>` in this plan — the card transform is plan 35-04.

Purpose: this is the core of the "buried hero / ticker blob" fix. It consumes the `ChipRail` primitive
(35-01) and `MarketRail`'s new `className`/default-closed behavior (35-02), and it owns all of
`Overview.tsx`'s header/grid/chrome region so plan 35-04 can own the positions region next wave without
a same-wave conflict on the same file.

Output: an Overview whose first phone screen answers "how am I doing" (KPI strip + hero) with no scroll
and no horizontal overflow, desktop pixel-identical.
</objective>
