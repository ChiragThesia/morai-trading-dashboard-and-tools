<objective>
Fix HIST-01 at its root cause: the SPX/SPXW OCC-root mismatch that makes every mixed-root calendar's back leg resolve to null, poison every journal row as a gap, and (post-OPS-01) silence snapshots entirely. RESEARCH traced this to four call sites that build/query a leg's OCC root from the single stored `calendars.underlying` for BOTH legs; this plan fixes all four with the shared `resolveRootCandidates` from plan 01 (try the stored root, then the sibling), plus the Pitfall-1 `mapSnapshotRow` source-drop that would otherwise hide a healed row from the exact UI this phase targets.

This is a bug fix, so each change lands behind a failing regression test first (tdd.md). No date-of-week classifier is introduced — the read/query sites trust whichever candidate root actually has a `contracts` row (RESEARCH §Alternatives Considered).

Purpose: Any leg with a usable mark in `leg_observations` becomes resolvable + greek-bearing, unblocking every downstream repair path.
Output: Fixed leg-resolution at all four sites + source-inclusive journal read.
</objective>
