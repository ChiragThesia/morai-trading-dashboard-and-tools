<objective>
Lay the two pure-function foundations the rest of Phase 40 builds on, and satisfy the docs-before-code rule for the new jobs this phase adds.

Per D-06, HIST-01's back-leg NaN is a SPX/SPXW OCC-root mismatch: the `calendars.underlying` column stores ONE root for both legs, but an End-of-Month back leg (e.g. `2026-11-30`) is exchange-listed under root `SPXW`, not `SPX`. Four resolution call sites construct the back leg's symbol from the single stored root, so the real, correctly-rooted quote in `leg_observations` is never looked up. This plan produces the one shared root-candidate function those four sites (plans 02/04) will reuse — no per-site date-of-week classifier.

Per HIST-05/D-07, the ~19-vs-13 rows/day inflation is a scheduling collision that produces two near-simultaneous `scheduled` rows with different wall-clock timestamps against a `(time, calendar_id)` composite PK. This plan produces the slot-rounding function plan 03 applies so the EXISTING `onConflictDoNothing` collapses same-slot writes with zero new dedup logic.

Per D-10 + CLAUDE.md rule 4, a new pg-boss job is "new tooling" — `docs/architecture/jobs.md`'s Job Catalog and its "historical gap rows are never backfilled" language become false once HIST-02/03/04 ship, so the docs land FIRST, ahead of any job code.

Purpose: Deterministic, tested primitives + a truthful job catalog, so plans 02–07 are pure reuse.
Output: `resolveRootCandidates`, `roundDownToRthSlot`, updated jobs.md.
</objective>
