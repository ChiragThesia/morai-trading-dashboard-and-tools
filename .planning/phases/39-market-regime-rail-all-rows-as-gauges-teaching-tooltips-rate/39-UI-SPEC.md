## Tooltip Copy (LOCKED, rev 3 ‚Äî condensed 2026-07-13)

**Rev 3 supersedes rev 2's four-part WHAT/WHY/BANDS/SOURCE layout.** User feedback after the
39-04 UAT: "The tooltips are too large and not easy to read. It should be quick and easy to
read not complicated." Every gauge row's ‚ìò tooltip now renders three lines, Hemingway style:

1. **WHAT** ‚Äî one punchy clause, ‚â§10 words, what the number is.
2. **WHY** ‚Äî one clause, ‚â§12 words, why it matters for SPX calendar trading.
3. **META** ‚Äî smallest/quietest line (`text-dim/70`), combining bands-or-range with source.

Copy below is FINAL ‚Äî executors render it verbatim, they do not paraphrase or invent
financial claims. This is a COMPRESSION of the rev-2 copy only ‚Äî every fact already existed
in the prior locked copy (see git history for the pre-rev-3 wording). The 4 existing regime
rows keep their server-provided `source`/`rationale` fields, appended below META, unchanged.
Rates and COT rows have no server-provided source field ‚Äî META bakes provenance in directly.

**`vix-term-structure` ‚Äî VIX/VIX3M**
- WHAT: VIX √∑ VIX3M ‚Äî near-term vs 3-month vol ratio

- META: Calm <0.90 ¬∑ warn 0.90‚Äì0.95 ¬∑ crisis ‚â•0.95

### Rates block (6 rows ‚Äî NEUTRAL, no bands, META bakes in source)

**Fed Funds (`DFF`)**
- WHAT: Fed's overnight bank lending rate ‚Äî sets everything else
- WHY: Sets the backdrop for options carry & hedge cost
- META: 0‚Äì8% range ¬∑ FRED DFF, daily

**SOFR (`SOFR`)**
- WHAT: SOFR ‚Äî repo rate that replaced LIBOR
- WHY: Gap vs Fed funds = money-market funding stress
- META: 0‚Äì8% range ¬∑ FRED SOFR, daily

**1M (`DGS1MO`)**
- WHAT: 1-month T-bill yield ‚Äî shortest US gov't rate
- WHY: Fast moves price in an imminent rate decision
- META: 0‚Äì8% range ¬∑ FRED DGS1MO, daily

**3M (`DGS3MO`)**
- WHAT: 3-month T-bill yield ‚Äî short leg of 10Y-3M
- WHY: Reflects near-term Fed policy expectations
- META: 0‚Äì8% range ¬∑ FRED DGS3MO, daily

**10Y-2Y (`T10Y2Y`) ‚Äî BANDED**
- WHAT: 10Y minus 2Y Treasury yield ‚Äî the curve slope
- WHY: Inverted = historical recession precursor, 6‚Äì12mo lead
- META: Calm >0 ¬∑ warn ‚â§0 ¬∑ crisis ‚â§‚àí0.50 ¬∑ FRED T10Y2Y, daily

**10Y-3M (`T10Y3M`) ‚Äî BANDED**
- WHAT: 10Y minus 3M Treasury yield ‚Äî a faster curve spread
- WHY: Inverting with 10Y-2Y = stronger recession signal
- META: Calm >0 ¬∑ warn ‚â§0 ¬∑ crisis ‚â§‚àí0.50 ¬∑ FRED T10Y3M, daily

## Gauge Component Contract

### Shared extraction

Extract the existing bullet-gauge track (the `<div role="meter">` markup + `axisPct`/
`clampedAxisPct` math in `RegimeBoard.tsx:114-126,217-243`) into
`apps/web/src/components/system/BulletGauge.tsx` (same directory as `ChipRail.tsx`/`Button.tsx`
‚Äî matches the existing extraction pattern). Presentational only:

```
BulletGauge({
  min, max, value,              // axis + true value (unclamped valuetext)
  variant: "banded" | "neutral",
  bandWarn?, bandCrisis?,       // required when variant="banded"
  markerColorClass,             // caller-computed Tailwind bg-* class
  ariaLabel, ariaValueText,
  testId,
})
```

`RegimeBoard`'s `Row` keeps its own label/value line and calls `BulletGauge` for the track.
New `RateGaugeRow` and COT row components do the same ‚Äî one shared track implementation,
three callers.

### Neutral vs. banded ‚Äî visual distinction (CONTEXT: "visually distinct... absence-of-verdict
legible")

- **Banded** (4 existing regime rows, + `t10y2y`/`t10y3m`): track shows the amber warn segment
  and red crisis segment behind the marker (unchanged `bg-amber/30`/`bg-down/30`). Marker color
  follows `BAND_CLASSES`/`MARKER_CLASSES` (txt calm / amber warning / down crisis) ‚Äî unchanged.
- **Neutral** (Fed Funds, SOFR, 1M, 3M): track is PLAIN `bg-line2` ‚Äî no band segments at all.
  Marker is always `bg-dim` (flat, muted) regardless of value ‚Äî never txt/amber/down, so a
  glance never mistakes "no verdict" for "calm."
- **Neutral, direction-tinted** (5 COT rows): track is plain `bg-line2` ‚Äî no band segments.
  Marker color follows sign: `bg-up` when net is long (‚â•0), `bg-down` when net is short (<0).
  This is the EXISTING long/short convention already in `CotCard.tsx` (`isLong` ‚Üí `bg-up`/
  `bg-down`), not a new verdict color and not amber-capable ‚Äî no warning tier is ever possible
  on a COT row. Keep the WoW `‚ñ≤`/`‚ñº` arrow + `fmtSigned`/`fmtMag` exactly as today, to the
  right of the track (mirrors the existing `regime-value`/gauge-marker layout: label+‚ìò left,
  value right, track full-width below).
  Size unification (Typography): `CotCard.tsx:90` today sets `text-[10px]` on the whole row
  container, cascading to the label, net, and WoW spans alike. Moving net/WoW to the shared
  11px secondary-value tier means removing that row-level size class and setting size per span
  instead ‚Äî label span gets an explicit `text-[10px]` (Row label tier, unchanged), net/WoW
  spans get `text-[11px]` (Value secondary tier) ‚Äî so the label doesn't drift to 11px too.

### Dense-mode labels

No new `SHORT_LABELS` entries needed. Every rate label (`Fed Funds`, `SOFR`, `1M`, `3M`,
`10Y-2Y`, `10Y-3M`) and COT label (`Dealer`, `Asset Mgr`, `Leveraged`, `Other rept`,
`Non-rept`) is already ‚â§10 characters ‚Äî all fit the dense rail width unshortened, same as
today. The layout change is the row shape (1-line label/value ‚Üí 2-line label/value + track),
not the labels themselves.

### A11y (meter role parity)

Every new row ‚Äî 6 rate rows + 5 COT rows ‚Äî gets the identical `role="meter"` contract already
on regime rows: `aria-valuenow` (clamped to `[min, max]`), `aria-valuemin`, `aria-valuemax`,
`aria-valuetext` (true, unclamped value + a plain-English state), `aria-label`. For neutral
rows, `aria-valuetext` states the value and "position" (e.g. `"4.33% ‚Äî position"`), never a
band word (no "calm"/"warning"/"crisis" vocabulary on a row with no verdict). For COT rows,
`aria-valuetext` states the signed value and WoW direction (e.g. `"+50.7K contracts ‚Äî
up 3.2K week-over-week"`).

### Tooltip layout (3-line structure, rev 3)

Same `TooltipProvider`/`Tooltip`/`TooltipContent` wiring as today (`RegimeBoard.tsx`), same
`‚ìò` `Badge` trigger. `TooltipContent` inner stack is 3 lines (condensed from rev 2's 4-line
WHAT/WHY/BANDS/SOURCE per user feedback ‚Äî "too large ... should be quick and easy to read"):

```
<div className="flex max-w-[15rem] flex-col gap-1 font-mono">
  <span className="text-[11px] text-txt">{WHAT}</span>
  <span className="text-[11px] text-dim">{WHY}</span>
  <div className="flex flex-col gap-0.5 text-[10px] text-dim/70">
    <span>{META}</span>
    {SOURCE}  {/* regime rows only: indicator.source + indicator.rationale, unchanged */}
  </div>

```

WHAT renders at `text-txt`, WHY one step quieter at `text-dim` (both 11px) ‚Äî the teaching
content, the point of this phase. META (bands, or bands+provenance baked in for rate/COT
rows) is quietest at `text-dim/70` (10px). Regime rows append the server's own
source/rationale below META unchanged ‚Äî the only rows with more than 3 visual lines, since
that provenance is dynamic per-instance data, not part of the condensed authored copy.
