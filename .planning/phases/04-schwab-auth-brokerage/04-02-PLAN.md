<objective>
Build the vendored Schwab OAuth client (token exchange + refresh grant, Basic-auth, Zod-parsed) and the Postgres `broker_tokens` repo that encrypts at rest with pgcrypto (key as a bound parameter, never in DB or logs), then the on-demand `refreshToken` use-case that ties them together. This is the AUTH-01/AUTH-02 core: the first end-to-end vertical from "call Schwab's token endpoint" through "encrypt and persist one token row" to "read it back decrypted."

Purpose: tokens are long-lived brokerage credentials — this slice owns the highest-risk security surface (encryption at rest, key handling, secret-in-logs avoidance). TDD here because every function has a defined I/O contract (msw for HTTP, testcontainers+pgcrypto for round-trip).
Output: oauth-client, broker-tokens Postgres repo + contract test, refreshToken use-case, new ports.
</objective>

  <files>packages/adapters/src/schwab/auth/oauth-client.ts, packages/adapters/src/postgres/repos/broker-tokens.ts, packages/core/src/brokerage/application/refreshToken.ts (+ their test files + contract harness)</files>
  <behavior>
    OAuth client (msw-backed, mirrors cboe.test.ts lifecycle):
    - buildAuthUrl(state) → "https://api.schwabapi.com/v1/oauth/authorize?client_id=...&redirect_uri=...&state=..." with the injected callbackUrl URL-encoded and the state param present.
    - exchangeCode(code) on HTTP 200 → ok({ accessToken, refreshToken, expiresIn }); request carries `Authorization: Basic base64(appKey:appSecret)`, `Content-Type: application/x-www-form-urlencoded`, body `grant_type=authorization_code&code=...&redirect_uri=...`.
    - refreshTokens(refreshToken) on HTTP 200 → ok(new SchwabTokens); body `grant_type=refresh_token&refresh_token=...`.
    - On HTTP 400 body {"error":"invalid_grant"} → err({ kind:"oauth-error", code:"invalid_grant", message }). Add a SEPARATE test for {"error":"invalid_client"} → code:"invalid_client" (RESEARCH A2 — test BOTH keys).
    - On network throw → err({ code:"network" }); on unparseable JSON → err({ code:"parse" }). Never throws across the port.

    broker_tokens Postgres repo (testcontainers + pgcrypto, mirrors leg-observations.contract.test.ts skip-when-no-Docker pattern):
    - writeTokens("trader", row) then readTokens("trader") → ok(row) with accessToken/refreshToken decrypted equal to the originals.
    - The raw stored column is ciphertext: a direct `SELECT access_token FROM broker_tokens` (bytea) does NOT equal the plaintext token.
    - readTokens for an unknown appId → ok(null).
    - readTokenFreshness() with a trader row issued now + a market row whose refresh_issued_at is 8 days old → ok({ trader:{status:"fresh"...}, market:{status:"AUTH_EXPIRED"...} }).
    - writeTokens is an upsert (onConflictDoUpdate on appId) — writing the same appId twice keeps one row.

    refreshToken use-case (in-memory twin + stub oauth, mirrors getStatus error-absorption):
    - Happy: read current tokens → call refreshTokens → on ok, writeTokens(rotated) → return ok(rotated).
    - invalid_grant: refreshTokens err invalid_grant → return err({ kind:"auth-expired", appId }); does NOT write.
    - No stored tokens for appId → return err (auth-expired or a typed not-configured error — choose one and test it).
  </behavior>
  <implementation>
    OAuth client: factory `makeSchwabOAuthClient(config: { appKey, appSecret, callbackUrl, fetch })` (PATTERNS oauth-client section). Precompute `Buffer.from(`${appKey}:${appSecret}`).toString("base64")`. Zod schema `SchwabTokenResponseSchema` = `z.object({ access_token: z.string(), refresh_token: z.string(), expires_in: z.number(), token_type: z.string().optional(), scope: z.string().optional() }).passthrough()` — safeParse before returning; map snake_case → camelCase SchwabTokens. Define `SchwabTokens` and `OAuthError` (kind:"oauth-error", code:"invalid_grant"|"invalid_client"|"network"|"parse") in the adapter and add `SchwabTokens`/`OAuthError`/`ForRefreshingToken` to brokerage ports.ts as needed. On non-2xx, read the JSON error body, map `error` field to the code. NEVER log the Basic header, tokens, or the error body verbatim (only kind/code).

    broker-tokens repo: factory `makePostgresBrokerTokensRepo(db, encryptionKey: string)`. WRITE via drizzle insert with `accessToken: sql`pgp_sym_encrypt(${plain}, ${encryptionKey})`` (and refresh) + `onConflictDoUpdate({ target: brokerTokens.appId, set: {...} })`. READ via `db.select({ accessToken: sql<string>`pgp_sym_decrypt(${brokerTokens.accessToken}, ${encryptionKey})`, ... })`. The key is ALWAYS a sql-template bound parameter — never `sql.raw`, never string interpolation (D-03, RESEARCH Pitfall 7). readTokenFreshness reads both apps' timestamp columns (no decryption needed) and composes `toAppTokenStatus`. catch → err({kind:"storage-error", message}) (leg-observations pattern). Never `console.*` the key or tokens — log only appId/timestamps.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app → Schwab token endpoint | appKey/appSecret (Basic auth) + authorization code / refresh token leave the process |
| app → Postgres | plaintext tokens cross into the DB and are encrypted with the injected key |
| Postgres wire protocol | the encryption key and token plaintext travel as bound parameters |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Information Disclosure | broker-tokens repo logs / errors | mitigate | Key + tokens NEVER passed to console.*; catch maps to {kind,message} with no secret payload; tests assert no plaintext in stored bytea |
| T-04-05 | Information Disclosure | pgcrypto key in query logs | mitigate | Key bound as sql-template `$n` parameter (never sql.raw / interpolation) per D-03; contract test exercises the real parameterized path |
| T-04-06 | Spoofing | OAuth token endpoint call | mitigate | Authorization: Basic base64(appKey:appSecret) on every token call; HTTPS endpoint pinned to api.schwabapi.com |
| T-04-07 | Information Disclosure | OAuth error responses (invalid_grant) | mitigate | Error body mapped to a typed code; only kind/code/message surfaced, refresh token never echoed |
| T-04-08 | Tampering | refresh-token rotation | mitigate | On 200 the rotated refresh token replaces the stored one via upsert; on invalid_grant nothing is written (no half-state) |
</threat_model>
