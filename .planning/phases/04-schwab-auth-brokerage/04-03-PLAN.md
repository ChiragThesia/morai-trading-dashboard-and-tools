<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → loopback listener | Schwab redirects the authorization code to 127.0.0.1 over HTTPS |
| CLI → Schwab token endpoint | code/refresh exchanged for tokens |
| CLI → Postgres | tokens encrypted + persisted |
| operator TTY → CLI | argv selects app + subcommand |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Spoofing / CSRF | OAuth state param | mitigate | state = crypto.randomUUID() sent in authorizationUrl and verified on callback (result.state === state) before exchange |
| T-04-10 | Tampering | loopback callback interception on localhost | mitigate | Listener bound to 127.0.0.1 only on the exact registered port; HTTPS (self-signed) per Schwab requirement; server shuts down immediately after one capture |
| T-04-11 | Information Disclosure | CLI stdout (setup/status/doctor) | mitigate | Only appId + timestamps + status printed; tokens, refresh tokens, app secrets, encryption key NEVER echoed |
| T-04-12 | Repudiation / config error | callback-URL mismatch | mitigate | doctor performs character-for-character compare of env callback vs registered value (D-06, Pitfall 1) and fails loudly |
| T-04-13 | Elevation / replay | 30-second auth code TTL | mitigate | exchangeCode called immediately after capture, before any other I/O (Pitfall 4) |
</threat_model>
