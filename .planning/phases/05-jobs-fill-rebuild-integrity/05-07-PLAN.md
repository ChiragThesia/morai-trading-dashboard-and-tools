<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Schwab fill data → sync-fills use-case | Fills ingested from the broker are untrusted input; OCC symbols are parsed (parseSchwabSymbol Result-branched) before matching |
| sync-fills → calendar_events / orphan_fills (Postgres) | Derived trade-ledger rows cross into the DB; idempotency enforced at the DB layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-17 | Tampering | duplicate-event injection via re-run | mitigate | fillIdsHash UNIQUE constraint + onConflictDoNothing makes the second sync-fills run a no-op (SC4); contract test asserts count unchanged on re-run |
| T-05-18 | Repudiation | orphan fills silently dropped | mitigate | Every unmatched/ambiguous fill is written to orphan_fills with a reason (D-05); use-case never drops a fill — proven by the orphan-parking test |
| T-05-19 | Input Validation | malformed OCC symbol / fill payload | mitigate | parseSchwabSymbol returns Result and is branched (no `as`); the handler Zod-parses job.data; a parse failure routes to orphan parking, not a crash |
| T-05-20 | Tampering | incorrect P&L (financial data) | mitigate | computePnl is property-tested (05-03); per-leg legBreakdown + net amounts asserted in the use-case test; numeric column avoids float drift (Don't-Hand-Roll) |
</threat_model>
