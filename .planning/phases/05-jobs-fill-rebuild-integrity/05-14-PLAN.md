<objective>
Close the round-2 BLOCKER and two small independent warnings that do not touch the fills
data-model:

- **CR-A1 (BLOCKER, MCP-02):** the WR-04 fix is half-applied. The HTTP route requires
  calendarId for rebuild-journal via `triggerJobBodyFor(name)`, but the MCP `trigger_job`
  tool validates with an inline `calendarId.optional()` schema and never calls the
  refinement, so `trigger_job {name:"rebuild-journal"}` with no calendarId still enqueues a
  null-keyed, un-deduplicated rebuild (the exact queue flood WR-04 was raised to close,
  reachable through the agent-driven MCP surface). Route the MCP tool through the same
  per-job refinement both adapters share.
- **WR-A3 (JRNL-01):** `hexToUuid` in syncTransactions drops hash nibble index 12
  (`"5" + h.slice(13,16)` skips `h[12]`), so two distinct `(activityId, legIndex)` keys that
  differ only at nibble 12 collide on the fills `id` PK and the second real fill is silently
  dropped by `onConflictDoNothing`. Use a contiguous, total nibble mapping so every input
  nibble contributes.
- **IN-A1 (info, JRNL-01):** `extractLastError` in job-runs still loops `Object.entries` to
  find the single known `"message"` key. Behavior-preserving cleanup to a direct narrow.

Purpose: stop the MCP queue-flood path and the silent fill-drop collision; both are
correctness bugs on the trade-ledger path.
Output: MCP parity for rebuild-journal validation, collision-free deterministic fill ids,
a small job-runs cleanup.

Note on IN-A2 (carry real commission/fees + real filledAt): NOT done here and NOT cheap.
The `BrokerTransaction` domain type (packages/core/src/brokerage/application/ports.ts) carries
no `time`, `commission`, or per-leg `fees` fields, and the transactions adapter does not
surface them. Threading real fees/time requires extending the brokerage domain type + the
Schwab adapter parse/map + a docs-first architecture change — out of scope for this gap
round. Tracked as a known limitation in the IN-A1 task's done-note and in STATE; realized
P&L stays fee-blind until a dedicated plan populates per-leg fees.
</objective>
