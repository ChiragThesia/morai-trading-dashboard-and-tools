<objective>
Fix Defect 1 (D-01, USER LOCKED): the payoff x-domain must fit the FULL tent — both
tails, both breakevens — for whatever is displayed, replacing the hardcoded 6900–7900 in
two independent places. This plan builds the domain PRIMITIVES: a pure
`computePayoffDomain` function, a domain-aware `repriceScenario`/`buildSpotGrid` data grid,
and a `PayoffChart` whose x-scale/ticks/wall-pinning/crosshair all read an explicit domain
prop. Screen wiring lands in 30-02.

Purpose: fixing only the chart scale (or only the data grid) still clips the curve
(Pitfall 1). Both must follow ONE computed `{min,max}` object.
Output: `payoff-domain.ts` (new), a parameterized `scenario-engine.ts` + `PayoffChart.tsx`.
</objective>
