<objective>
Fix defect DEFECT-1: the Risk Profile chart's γflip / put wall / call wall marker labels
collide into unreadable text when the levels cluster within ~60 SPX pts (0DTE-heavy days;
real repro 2026-07-10: flip 7488 / putWall 7500 / spot 7544 / callWall 7550 within 62 pts).

Per the UI-SPEC design contract (Component Spec 1), the chosen KISS strategy is to DELETE
the in-chart text label entirely and keep only the dashed vertical line — the label has no
collision surface if there is no text. The exact numeric value already lives in three other
places (Overview "Key levels" panel, Analyzer ScenarioStrip level row, crosshair tooltip),
so nothing is lost. Off-domain walls (a real case since Phase 30 anchors the domain on
strikes/spot/breakevens, not GEX walls) render a single-glyph edge arrow in a fixed vertical
lane per series, provably non-overlapping by construction. The Overview legend gains three
per-series wall swatches so the color→meaning mapping survives the label removal.

NOTE ON SOURCE CONFLICT: RESEARCH.md sketched an `assignLabelRows` stagger; the UI-SPEC
supersedes it on visual treatment (delete labels + fixed-lane arrows). RESEARCH remains
authoritative for file:line facts and the jsdom test strategy (assert lane/anchor
assignment, never pixel measurement).

Purpose: Chart reads clean on the tightest-clustered live day; KISS chrome per user request.
Output: Reworked PayoffChart marker layer (shared by Overview + Analyzer) + Overview legend.
</objective>
