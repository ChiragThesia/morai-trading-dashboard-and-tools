# Phase 35.1 — UI Design Contract: Mobile Overview, Dedicated Component Tree

> Phase 35 reflowed the desktop Overview below 1024px. The user's verdict on a real
> phone: still garbage. This spec designs the mobile Overview **from scratch** as a
> dedicated mobile-only component tree — hero P&L first, full-bleed chart with one
> control row, refined position cards, purpose-built market section — while the desktop
> (≥1024px) render output stays byte-identical. Same data hooks, same scenario engine,
> same MORAI tokens; only the view tree is new.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (already initialized — `apps/web/components.json`) — no `shadcn add` this phase |

## Decisions

| ID | Decision |
|----|----------|
| **D-01** | **Dedicated mobile tree, single top-level branch.** New directory `apps/web/src/screens/overview-mobile/`. `Overview.tsx` becomes a thin switch: `useIsDesktop()` → `<OverviewDesktop/>` (today's JSX, renamed in-file, byte-identical DOM) or `<OverviewMobile/>` (new tree). Only ONE tree mounts at a time — one PayoffChart, one live-stream consumer, no CSS dual-render of heavy content. |
| **D-02** | **Shared model hook, duplicated views.** All non-trivial state/derivation currently inline in `Overview()` (data hooks, live stream, calendar build + IV calibration, scenario + domain, toggles, date control, excluded set, selected row, verdict maps, exits states, freshness, bookPnl, railGreeks) extracts to `useOverviewModel()` (new file, `apps/web/src/screens/overview-mobile/useOverviewModel.ts` or `hooks/`). Both trees call it. View JSX is duplicated per tree (sanctioned); logic is never duplicated. Module helpers `fmtGammaCompact`, `latestMacroValue`, `keyLevelsFor`, `buildRows` move/export alongside. Breakpoint-cross (resize past 1024px) remounts the tree and resets UI state (toggles, exclusions) — accepted, phones don't cross it. |
| **D-03** | **Hero block = BOOK P&L.** Value source is the SAME `bookUnrealizedPnl(positions)` that feeds today's BOOK chip — no new P&L path this phase (a live-tick hero is a possible follow-up, not scope). 32px mono bold, sign-colored TEAL/CORAL. Context line beneath: `SPX {spot} · VIX {vix} · γ {AMPLIFY\|DAMPEN}`. |
| **D-04** | **No ticker strip on mobile. At all.** The 4 priority values live in the hero (book = hero number; SPX/VIX/regime = context line). The 6 secondary metrics (0DTE γ, γ flip, VVIX, Fed funds, 10y−2y, COT lev) relocate into the MARKET section as stat grids (γ flip already lives in Key levels — not repeated). `ChipRail` and `MetricChip` are not mounted anywhere in the mobile Overview tree. |
| **D-05** | **Chart full-bleed, one control row, no legend.** Mobile chart section has NO Panel wrapper, NO panel title, NO legend row, NO freshness chips above the chart. Exactly one control row above the chart: `‹ [date] ›` stepper + right-aligned `@ exp` toggle + `⋯` overflow button. Fan/Walls/Profit-zone toggles and "Today" reset live inside a `⋯` Dialog (reuses shadcn Dialog, same state/handlers). Freshness reduces to one 9px muted caption line BELOW the chart with a single worst-of dot. |
| **D-06** | **PayoffChart additive props (internals untouched).** Two new optional props, both defaulting to today's behavior so desktop Overview + Analyzer call sites are unaffected without edits: `showBePills?: boolean = true` (mobile passes `false`; BE-on-axis labels — the user-loved Phase 33 feature — are unconditional internals and remain) and `aspectRatio?: number = SVG_W / SVG_H` (mobile passes `1.3` → ~300px tall at 390px). No other chart file changes. |
| **D-07** | **PositionCard re-hierarchy.** Row 1: label + IV-n/a badge + VerdictChip left, **unreal P&L right-aligned prominent** (16px mono bold, sign-colored). Row 2: muted meta (expiry · DTE · net val). Expand: Δ/Γ/Θ/Vega grid + `VerdictDetailBody` when a verdict exists (desktop parity — new `verdictObservedAt` prop). Checkbox-include (44px hit area) and tap-to-expand behaviors unchanged. PositionCard is mobile-only (sole consumer is the card list), so restyling it cannot regress desktop. |
| **D-08** | **Market section = purpose-built mobile forms.** Order: Key levels (survives as-is — it works) → GEX stat grid (net γ/1% + 0DTE γ) → Net book greeks 2×2 (as-is) → Macro stat grid (VVIX / Fed funds / 10y−2y / COT lev) → collapsed `<details>` disclosure containing `RegimeBoard dense` + `CotCard` + System health (reuses `MarketRail` component verbatim). **`GammaProfile` and `GexBars` do not render on mobile** — they are desktop-only visualizations; their headline numbers (walls, flip, net γ) are all present in the lists/grids above. |
| **D-09** | **Catch #24 compliance.** Every structural open/closed state uses real attributes/state: `MarketRail`'s `<details>` keeps its `open={isDesktop \|\| undefined}` mechanism (false in the mobile tree → closed by default, user-toggled); the `⋯` overflow uses Dialog (portal state); card expand uses React state + conditional render. No CSS-reveal of closed disclosure content anywhere. |
| **D-10** | **jsdom defaults to the mobile tree.** `useIsDesktop` returns `false` under jsdom, so tests render `OverviewMobile` by default; desktop assertions stub `window.matchMedia` (the Phase 35 pattern). Existing `Overview.test.tsx` desktop-branch tests MUST be migrated to the stub in the same plan that lands the switch — this is expected churn, not regression. A desktop DOM guard test (matchMedia-stubbed render, structural assertions) protects byte-identity. |
| **D-11** | **Positions footer total.** One mono summary line under the card list — `Net {usd} · {unreal signed} · {n}/{m} included` — computed with the same `resolveLivePositionRow` over included legs the desktop Net row uses (shared lib fn, view-level call). |
| **D-12** | **Desktop dead-branch cleanup (final task).** Once the mobile tree renders below 1024px, the Phase 35 `lg:hidden` branches inside the desktop path (PillHeader priority row + secondary ChipRail, PositionsTable card list, payoff-chart `-mx-3` bleed wrapper's mobile arm) are unreachable-visible (`display:none` at every width the desktop tree mounts). Remove them in a dedicated last task, gated by the 1440px screenshot tripwire. `ChipRail` itself stays (PayoffControls uses it on desktop/Analyzer). |
| **D-13** | **No chart highlight from mobile card taps.** Desktop row hover/select dims the net-book curves and overlays the single position. On mobile the chart is usually scrolled off when tapping a card, and there is no hover; the mobile tree passes `highlightedPositionId={null}` / highlighted curves `null`. Card tap = expand only. |
| **D-14** | **MarketRail summary copy.** `<summary>` text changes from `Market` to `Regime · COT · health` — it now sits INSIDE a section already headed "MARKET", so "Market ▸" would be redundant. The summary is `lg:hidden` (invisible on desktop), so this is a mobile-only visible change. |

### Open questions — resolved inline (no user available)

1. *Hero number: live-tick total vs polled bookPnl?* → Polled `bookUnrealizedPnl` (D-03): it is the exact number the user already reads as BOOK; introducing a second P&L derivation is new logic risk for zero requested value.
2. *Where does "Today" go when the control row is too tight at 390px?* → Into the `⋯` Dialog as its first action; the date input gets a violet ring when projected (≠ today) so the non-today state is visible in the row (D-05, §MobileRiskPanel).
3. *Do Dealer-γ / GEX-by-strike charts get "compact mobile versions"?* → No. Ground truth shows they are unreadable at 390px; a smaller version is still axis soup. Headline numbers only (D-08); full charts remain a desktop feature.
4. *State persistence across the 1024px cross?* → Not preserved (tree remount, D-02). Phones never cross it; a resized desktop window re-derives everything from the shared query cache in one render.

## Component Specifications

### 1. `Overview.tsx` — the switch (D-01/D-02)

```tsx
export function Overview(): React.ReactElement {
  const isDesktop = useIsDesktop();
  return isDesktop ? <OverviewDesktop /> : <OverviewMobile />;
}
```

- `OverviewDesktop` = today's `Overview` body renamed in the same file, JSX untouched (except D-12 cleanup in the final task). Byte-identical DOM at ≥1024px.
- `OverviewMobile` imported from `screens/overview-mobile/OverviewMobile.tsx`.
- Both call `useOverviewModel()` — the extracted hook returning (names indicative, planner may adjust): `{ positions, rows, spot, gex, macroValues: { vix, vvix, dff, curveSlope }, cotLev, bookPnl, regime, zeroDte, liveGreeks, liveStatus, liveBadgeProps, calendarPositions, ivNaByRowKey, verdictByRowKey, unlinkedVerdicts, exits: { snapshot, isPending, isError, refetch }, scenario, payoffDomain, positionSetSignature, excludedFromT0Count, toggles, handleToggle, dateControl, bounds, excluded, handleToggleExcluded, selectedRowKey, handleSelectRow, hover: { highlightedRowKey, handleHoverRow, highlightedScenario }, freshness: { gexFresh, gexAsOf, gexAgeMs, markFresh, markAsOf, markAgeMs }, railGreeks }`.
- The hover/highlight slice is consumed by desktop only (D-13).
- `keyLevelsFor`, `fmtGammaCompact`, `latestMacroValue`, `buildRows` exported for the mobile tree — never re-implemented.

### 4. `PayoffChart` — additive props only (D-06)

```ts
/** Hide the BE readout pill row above the plot (mobile passes false; BE axis labels
 *  are unaffected). Default true — desktop Overview and Analyzer render exactly as today. */
showBePills?: boolean;   // default: true
/** CSS aspect-ratio for the ChartContainer. Default SVG_W / SVG_H (≈2.13) — today's exact
 *  value. Mobile passes 1.3 for a taller phone-width plot. */

```

Implementation contract: `showBePills === false` skips the `data-testid="be-pills"` block; `aspectRatio` feeds `style={{ width: "100%", aspectRatio: String(aspectRatio) }}` on the existing `ChartContainer`. Nothing else in the file changes — z-order, marks, grid, tooltip, scales all frozen.

### 5. `PositionCard` — re-hierarchy (D-07, edit in place)

New/changed props: `+ verdictObservedAt: string | null` (for `VerdictDetailBody`). All existing props unchanged.

Layout contract (classes/tokens exact):

- Card surface: `rounded-lg bg-raise/30 p-3 ring-1 ring-line transition-opacity` (`opacity-40` when excluded — unchanged). The subtle `bg-raise/30` (over today's `bg-transparent`) is the "card, not table row" cue; no new color, existing token at existing opacity step.

- **Row 1** (`flex items-center justify-between gap-2`): left `flex min-w-0 items-center gap-1.5` → label (`font-display text-sm font-bold text-txt`), IV n/a Badge (unchanged), `VerdictChip` (unchanged component); right → unreal: `font-mono text-base font-bold tabular-nums` + `signClass(unreal)` (`text-dim` "—" when null).
- **Row 2** (`mt-1 font-mono text-[10px] text-dim truncate`): `{expiry.line1} · {expiry.line2} · {usd(netVal)}`.
- The old two-`Stat` Net val/Unreal row is REMOVED (net val → row 2 meta; unreal → row 1 focal).
- **Expanded** (`mt-2 border-t border-line/40 pt-2`): existing 4-col greeks grid, then `verdict !== null && <VerdictDetailBody row={verdict} observedAt={verdictObservedAt ?? ""} />` in `mt-2`.
- Expand stays un-gated by verdict (catch #23 — greeks are mobile's only greeks surface).

### 6. Positions section (in `OverviewMobile.tsx`)

- Heading row: `flex items-center justify-between gap-2 mb-2` → `SectionLabel` `Positions` + `LiveStatusBadge {...liveBadgeProps}` + existing `Exit rules ▸` `Dialog` trigger (same classes as desktop trigger; trigger is comfortably tappable at its text size + `py` — acceptable, it matches the desktop affordance and sits alone at the row's end).
