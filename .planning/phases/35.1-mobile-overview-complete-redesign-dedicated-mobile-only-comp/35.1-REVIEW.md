## Warnings

### WR-01: Mobile shows Net-book greeks computed at the 5800 spot fallback when no GEX snapshot exists

**File:** `apps/web/src/screens/overview-mobile/MobileMarketSection.tsx:86-92` (with `apps/web/src/screens/overview-mobile/useOverviewModel.ts:392,537-540`)
**Issue:** The model computes `railGreeks = netGreeksForLegs(allLegs, spot)` where
`spot = gex?.spot ?? 5800`. The desktop `GexRail` early-returns the entire rail
(including the Net-book-greeks grid) when `gex === undefined`, so the fallback-spot
greeks are never displayed. `MobileMarketSection` gates only key-levels/gamma on
`gex === undefined` and renders the Net-book-greeks 2×2 unconditionally — on a
GEX-cold-start the mobile screen shows plausible-looking Δ/Γ/Θ/Vega numbers priced at
spot 5800 against a real SPX near 6900+, with no unavailability cue. Pre-35.1, mobile
viewports rendered the desktop tree and never saw these numbers in that state, so this
is a phase-introduced divergence. UI-SPEC §7 sanctions the layout (block 4 sits outside
the gex guard) — this is the spec's own bug, same class as catch #23.
**Fix:** Gate the grid on the same condition the numbers depend on:
```tsx
{gex !== undefined && (
  <>
    <SectionLabel tone="dim">Net book greeks</SectionLabel>
    <div className="grid grid-cols-2 gap-2">…</div>
  </>
)}
```
or render `—` per Stat when `gex === undefined` (matching the hero's per-segment `—`
rule from the Copywriting Contract).

**File:** `apps/web/src/screens/Overview.tsx:132`

### IN-04: PositionCard nests `<div>` blocks inside a `<button>`

**File:** `apps/web/src/components/PositionCard.tsx:73-118`
**Issue:** The expand button's children (row-1 flex div, meta div, expanded grid div)
are flow content; the HTML button content model permits phrasing content only.
Browsers and the a11y tree tolerate it, and no interactive element is nested (verified),
but it is technically invalid markup.

`role`/keyboard handling, or accept as-is (common pattern, no observed breakage).

design and safe (single-consumer invariant verified; react-query data survives) — only
observable on iPad rotation or desktop window resize across 1024px.
**Fix:** None required; documenting the accepted trade-off.
