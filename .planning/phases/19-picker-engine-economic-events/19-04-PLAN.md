<objective>
Build the complete economic-events data path (PICK-03, D-12/D-13): a FRED `release/dates` HTTP adapter
for CPI + NFP unioned with a maintained FOMC seed into one `EconomicEvent[]`; an in-memory twin; a new
`economic_events` Postgres table (plain `date` column, composite PK) + repo; and one shared contract
suite exercised against both implementations. This is the phase's single research-gated deliverable.

Purpose: The forward-looking calendar the scoring engine reads for per-leg event-window flags.
Output: HTTP adapter, memory twin, Postgres table + repo, shared contract suite.
</objective>

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-19-07 | Denial of Service / data corruption | malformed FRED JSON | high | mitigate | New FredReleaseDatesSchema safeParse → err on any shape mismatch; job/caller logs and moves on, never throws unguarded |
