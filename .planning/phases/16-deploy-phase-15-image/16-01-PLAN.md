<objective>
Close the live security exposure discovered during research and establish exact pre-deploy
ground truth, so the deploy in Plan 02 is a verify-then-force operation rather than a blind push.

Two things must be true before any `railway up` runs:
1. The sidecar service has no public domain (GW-05: the sidecar owns Schwab tokens and must be
   reachable only via `SIDECAR_URL` on the private Railway network). A public domain

   post-deploy proof (Plan 02) and smoke check (Plan 03) compare against a known baseline and do
   not misattribute pre-existing noise as a deploy regression (RESEARCH Pitfall 5).

This plan honors D-03 (verify-before-deploy, confirm migration parity) and D-05 (zero code — this
plan touches only Railway/Vercel/verification state, no `packages/` or `apps/*/src/`).

Purpose: Remove a real information-disclosure exposure and prevent both a missed and a
misattributed deploy signal.
Output: Sidecar confirmed domain-less; a pre-deploy baseline + readiness record in the SUMMARY.
</objective>
