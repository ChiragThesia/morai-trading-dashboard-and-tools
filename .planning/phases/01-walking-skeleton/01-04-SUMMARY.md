## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] CalendarsRepo type from __contract__ imported by production files**
- **Found during:** Task 2 typecheck — contract file excluded from tsconfig but production files imported from it
- **Issue:** `makeMemoryCalendarsRepo` and `makePostgresCalendarsRepo` originally returned `CalendarsRepo` type from `__contract__/calendars.contract.ts`. The contract file uses vitest imports and is excluded from tsconfig emit; this created a TypeScript TS6307 error
- **Fix:** Each adapter defines its own return type (`MemoryCalendarsRepo`, `PostgresCalendarsRepo`). Production code doesn't import from `__contract__`. Index.ts removed re-export of contract harness (accessible via direct import in test code)
- **Files modified:** `packages/adapters/src/memory/calendars.ts`, `packages/adapters/src/postgres/repos/calendars.ts`, `packages/adapters/src/index.ts`
- **Commit:** d51b1fd

**2. [Rule 1 - Bug] adapters→adapters intra-package imports not allowed by boundaries rule**
- **Found during:** Task 2 lint — intra-package relative imports (db.ts, schema.ts, migrate.ts importing each other) flagged by boundaries/dependencies
- **Issue:** The `boundaries/dependencies` rule only allowed `adapters → ["core", "shared"]` but not adapters→adapters intra-package imports
- **Fix:** Added `"adapters"` to the allow list for the adapters element type (same pattern as core→core from Plan 03)
- **Files modified:** `eslint.config.js`
- **Commit:** d51b1fd

**3. [Rule 1 - Bug] ESLint project:false block didn't cover __contract__/test/vitest.d.ts files**
- **Found during:** Task 2 lint — `Parsing error: "parserOptions.project" has been provided but file not found in any project`
- **Issue:** The lint config's project:false block covered `*.test.ts` and `vitest.config.ts` but not `__contract__/**`, `vitest.d.ts`, or `test/**` (globalSetup)
- **Fix:** Extended the `files` glob in the project:false ESLint block; also disabled type-aware rules (`no-floating-promises`, etc.) for that block
- **Files modified:** `eslint.config.js`
- **Commit:** d51b1fd

**4. [Rule 3 - Blocking] vitest.config.ts poolOptions deprecated in Vitest 4**

- **Issue:** Vitest 4 moved `poolOptions.forks.singleFork` to top-level `singleFork: true`
- **Fix:** Updated vitest.config.ts to use `singleFork: true` at test top level
- **Files modified:** `packages/adapters/vitest.config.ts`
- **Commit:** d51b1fd

**5. [Rule 2 - Missing] drizzle-kit needed at repo root (not just adapters package)**

- **Files modified:** `package.json`, `bun.lock`

- **Files modified:** `packages/adapters/src/postgres/repos/calendars.contract.test.ts`, `packages/adapters/src/postgres/migrate.idempotent.test.ts`
- **Commit:** d51b1fd

## Threat Surface Scan

**T-01-08 (Tampering — Migration corruption from pooled connection):** Mitigated. `runMigrations` creates a dedicated `postgres(url, { max: 1 })` client, never reuses the API pool. Closes with `await client.end()` in `finally` block.

**T-01-09 (Tampering — SQL injection via Drizzle):** Mitigated. All queries use Drizzle's query builder (parameterized). The only raw SQL is `sql\`SELECT 1\`` for pingDb and the partial index expression in schema — both are static strings, no user input interpolation.

**T-01-10 (DoS — Container-per-worker exhausting CI Docker):** Mitigated. `globalSetup` starts ONE `postgres:16` container, provides the URI via `context.provide('dbUrl', uri)`. `singleFork: true` ensures all test files run in one worker process.

No new security-relevant surfaces introduced beyond the plan's threat model.
