AUI-07 (D-07, explicit override of Phase 38's snapshot-spot lock): make the Analyzer's spot/marks
consume the live sidecar stream with an honest stale-fallback, and mount the LiveStatusBadge on both
the desktop and mobile trees. AUI-06: the mobile chart chrome row gets the same badge.

This is a direct port of Phase 38's LIVE-04 seam (already in production for Overview) — one
`useLiveStream()` consumer added to `useAnalyzerModel`, one live-aware `spot` ternary replacing the
current `snapshot?.spot ?? 0`, one `liveBadgeProps` object mirrored from `useOverviewModel`. The
existing `params`/`payoffDomain`/`scenarioResult` memo chain already closes over `spot`, so the live
value reaches `PayoffChart` automatically — only the source of `spot` changes.

Purpose: the T+0 payoff marker reflects the live SPX tick, not a stale 30-min cohort, while the
score/verdict stay point-in-time honest.
Output: live-aware `useAnalyzerModel` + its regression test; LiveStatusBadge mounted on both trees.
</objective>

| Boundary | Description |
|----------|-------------|
| sidecar SSE → browser | Live spot/tick frames cross into the client render (already Zod-validated in useLiveStream) |
| snapshot vs live | The score/verdict path (snapshot) must never be mutated by the live-tick path |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-41-01 | Tampering (of trust) | useAnalyzerModel spot ternary | medium | mitigate | Double-gate `liveStatus === "live" && liveSpot !== null` — a frozen live value after disconnect never shows as fresh (catch #26); regression-tested in Task 1 |
| T-41-03 | Tampering | scoring/verdict path | medium | mitigate | `spot` stays scoped to the params/payoffDomain/scenarioResult memo chain only; candidate.score/breakdown/theta/vega/debit take zero live input — enforced by construction, verified by not threading liveSpot into WhyPanel/EntryExitPlan/hero |
| T-41-DoS | Denial of Service (client) | malformed SSE frame | low | accept | useLiveStream already safeParse-and-drops every event; this plan adds a consumer, not a parser — inherits existing hardening |
| T-41-SC | Tampering | npm installs | n/a | accept | Zero package installs this phase (UI-SPEC Registry Safety) — no supply-chain surface |
