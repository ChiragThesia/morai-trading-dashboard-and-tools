# Phase 41 — UI Design Contract

> Visual and interaction contract for the Analyzer cleanup: ranked candidate table, verdict
> hero, sticky layout, trading-precision numbers, paste/term-structure polish, mobile parity,
> and live sidecar data. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

No new palette, no new dependency, no new shadcn component. This phase restructures
`Analyzer.tsx` (desktop) and extends `analyzer-mobile/` (Phase 36 tree) using primitives that
already exist in this codebase: `Panel`/`PanelHeading`/`Button`/`SectionLabel` (system),
`LiveStatusBadge` (Phase 20/38), the existing `scoreStatus()` classifier, and a plain HTML
`<table>` (Overview.tsx's own precedent — zero new dependency). All decisions below are either
locked in `41-CONTEXT.md` or resolved here as Claude's Discretion per that document.

## Table Contract (AUI-01, D-01)

Replaces the 17-card scroll rail (`CandidateRail`'s `<CandidateCard>` list) with one compact
`<table>` inside the existing left-column `Panel`. Row click drives the exact same
`onSelect`/`selectedId` state the cards drove — zero change to `useAnalyzerModel`.

### Columns (locked, left to right)

| Column | Source field | Format | Sortable |
|--------|-------------|--------|----------|
| Score | `candidate.score` | `Math.round(score)`, bold, color by tier (see Color) | Yes — default sort, desc |
| Calendar | `candidate.name` | rendered verbatim (already a formatted string, e.g. `"7525P 2026-08-06 / 2026-08-10"` — no new date formatter) | No |
| Debit | `candidate.debit` | whole dollar, `$${Math.round(debit)}` (see Number Rounding Contract — NOT `exactAbs`/`usd()`) | Yes |
| Θ/d | `candidate.theta` | `${theta>=0?'+':''}${theta.toFixed(1)}/d`, `text-up`/`text-down` by sign (existing convention, unchanged) | Yes |
| Event | `candidate.frontEvents[0] ?? candidate.backEvents[0] ?? null` | amber pill, event name + `⚡`; `text-dim` `"—"` when both arrays are empty; `+N` suffix when more than one event exists across both legs | No |
| ⊕ | `combinedIds.has(candidate.id)` | icon-only `Button variant="toggle" tone="amber" size="xs"` with `aria-label={\`Combine ${candidate.name}\`}` (rev 2, ui-checker Dim-2; the ⊕ column `<th>` carries a visually-hidden "Combine" label) (existing per-card Combine affordance, moved to its own narrow column since Combine must stay triggerable on rows the user has NOT selected — Copy stays detail-pane-only, see below) | No |

Pasted rows (`pastedCandidates`) render pinned above scored rows in paste order, unchanged
ordering. Score cell shows the existing "PASTED" pill instead of a number; that row's rightmost
cell additionally carries the existing `×` remove button next to the ⊕ toggle.

### Row states

- **Default**: `border-b border-line/60`, `text-txt` — Calendar/Debit/Θ/Event cells at Row-value
  tier (11px).
- **Hover**: `hover:bg-line/40` (matches the existing rail-empty-state hover convention).
- **Selected**: `bg-violet/[0.06]` + `border-l-2 border-l-violet` on the row — the exact tint
  `CandidateCard.tsx` already uses for `selected`, applied to a `<tr>` instead of a card border
  (a full border doesn't render cleanly on table rows; the left accent bar is the standard
  substitute and needs no new token).
- **Not-scored (pasted, unscored)**: Score cell reads "PASTED" (violet pill, existing markup);
  Debit/Θ cells render `—` (a pasted-but-unscored calendar has no engine debit/theta — never a
  fabricated `$0`).

### Selection linkage

Clicking anywhere in a row except the ⊕/× cell calls `onSelect(candidate)` (`stopPropagation` on
the action cell, exact pattern `CandidateCard.tsx` already uses on its Combine/Copy buttons).
Selection state must be visually explicit per AUI-01 — the violet row tint above IS that
explicit state; no separate "selected" badge needed.

### Sort affordance

Score/Debit/Θ column `<th>` are `cursor-pointer`, `hover:text-txt`, carry `aria-sort`
(`"ascending"`/`"descending"`/`"none"`) and a trailing `▲`/`▼` glyph on the currently active sort
column only. Clicking a sortable header cycles: default (score desc) → asc → desc → back to
that column's desc, one click per direction change (no 4-state toggle — 2 states per column plus
"not this column" is enough; Calendar/Event never become the sort key). Default view on load is
unchanged (`sortedCandidates`'s existing score-desc order) — sort state is client-only UI state,
never persisted, never affects `pastedCandidates`' pinned-above ordering.

### Bounded height (kills the "17-card wall" and the "dead columns" defect together)

The table's `Panel` wrapper gets `max-h-[70vh] overflow-y-auto`; the `<thead>` gets
`sticky top-0 z-10 bg-panel`. This single change is what makes AUI-03's "no dead columns, page
height content-driven" true: today's 3,274px page height comes entirely from the 17-card rail
stretching the CSS grid row (`grid-cols-[300px_minmax(0,1fr)_330px]`) to its height, leaving the
center/right columns' ~600px of real content beside ~2,600px of empty space. Capping the LEFT
column's own height (not the grid row) means the grid row is only ever as tall as the taller of
"table capped at 70vh" vs. "center+right natural content height" — which today is the
center+right stack (payoff chart + term-structure chart + WHY/plan panels, roughly 800–900px).
No `position: sticky` on the center/right columns is needed (simpler, and avoids the known
sticky-inside-CSS-grid interaction quirks); this is the "or equivalent" branch of D-03's own
wording.

## Verdict Hero Contract (AUI-02, D-02)

Replaces the 11-chip flat row (`ScoringMethodologyPanel`) and its bare debug line with one
headline + three labeled factor groups, exactly matching the user-approved preview in
`41-CONTEXT.md`. Renders in the same `analyzer-scorecard-wrapper` slot, above the 3-column grid.

### Verdict word derivation (evidence-honest — reuses the existing classifier, invents nothing)

```
scoreStatus(candidate.score)   // apps/web/src/screens/analyzer-mobile/useAnalyzerModel.ts:74
  icon "✓" (score ≥ 66.7) → "FAVORABLE", text-up
  icon "~" (score ≥ 33.3) → "CAUTION",   text-amber
  icon "✗" (else)          → "SKIP",      text-down
```

This is the SAME ✓/~/✗ two-thirds/one-third split already applied per-criterion in
`scoreStatus()`, applied once more to the already-engine-computed overall `candidate.score` — no
new threshold, no fabricated confidence number, per D-02's evidence-honesty law.

### Headline row

```
{icon} {WORD}     score {Math.round(score)}/100     Θ {theta≥0?'+':''}{theta.toFixed(1)}/d
```

`{WORD}` at Verdict-word tier (16px 600 semibold, tier color — rev 2). `score …/100` and `Θ …/d` at Headline-value
tier (13px 600 semibold mono — rev 2); Θ keeps its existing sign color (`text-up`/`text-down`) independent of the
verdict-word tier color — a positive score-tier candidate can still show a red Θ if theta is
negative (never let one signal silently overwrite another). AH-session badge (existing
`SESSION` / "AH — indicative" amber pill) renders immediately after the headline row when
`marketSession === "after-hours"` — same copy, same amber token, unchanged from today.

### Factor groups (locked mapping — verbatim from the user-approved preview)

| Group | Criteria (in display order) |
|-------|------------------------------|
| EDGE | `fwdEdge`, `slope`, `vrp` |
| RISK | `eventAdjustment`, `beVsEm`, `debitFit` |
| FIT | `gexFit`, `deltaNeutral`, `thetaVega` |

Three `SectionLabel`-tier (10px) group headers in a row (`grid grid-cols-3 gap-4`), each followed
by its 3 factor rows stacked (`gap-y-1`). Each factor row is `label ✓/~/✗ NN%` at Row-value tier
(11px) — identical lookup-by-criterion-name, identical guard handling (fwdEdge-null,
gex-context-not-ok, events-context-not-ok → `n/a` at zero width/no bar), and identical
`CHIP_LABELS` short names the existing chips already use — this section is a re-layout of
existing data, not new data. A criterion missing from `candidate.breakdown` (pre-registry
snapshot) renders nothing in its slot — same as today's chip-omission behavior.

θ GATE (the raw net-theta pass/fail chip) is retired as a separate element — its information is
now the headline's Θ value (colored by sign), so it is never duplicated as both a headline number
and a redundant chip.

### Footer (calibrating + dropped-quotes + as-of — quiet, D-02's "never a floating row")

One `text-dim text-[10px]` line beneath the three groups, omitted entirely when both its
sub-parts are empty:

```
{CALIBRATING context entries joined by " · ", if any}   {gate-drop counts, if any}   as of {HH:MM} · {source}
```

Reuses `CandidateCard.tsx`'s existing `formatAsOf(observedAt)` verbatim for the "as of" segment —
this is the as-of provenance AUI-07 requires on every displayed number; putting it once in the
hero footer (rather than repeating a staleness dot on every table row) keeps the table scannable
while keeping the score's provenance one glance away. Existing `checklist-gate-drops` and
`checklist-experimental` copy strings carry over unchanged.

### Not-scored candidate (pasted, `breakdown.length === 0`)

Renders ONLY `PASTED_NOT_SCORED_NOTE` at Row-value tier — no verdict word, no score, no groups
(catch #23: never a hollow/fabricated verdict). No candidate selected → renders nothing (matches
`MobileScorecard`'s existing `candidate === null` → `null` convention).

## Number Rounding Contract (AUI-04, D-04)

Display-layer only — never mutates a stored/computed value, per D-04's own wording.

| Field | Rule | Current offender / fix |
|-------|------|------------------------|
| Dollars (debit, book debit) | Whole dollar, `Math.round()` | `exactAbs(candidate.debit)` today prints every decimal (`$652.2xxxxx` class of bug) — **do NOT reuse `exactAbs`/`usd`/`signedUsd` from `position-format.ts` for Analyzer dollars.** Those helpers are the Journal/Overview "show the exact broker value" contract (their own locked UI-SPECs) — reusing them here would silently regress that law. Analyzer needs a LOCAL `Math.round()`-based formatter (inline one-liner is enough — ladder rung 6, no new shared helper file justified for one screen) |
| Greeks: vega, delta | ≤2dp, `.toFixed(2)` | `exactAbs(candidate.vega)` is the literal `+61.9536112` example named in AUI-04 — replace with `candidate.vega.toFixed(2)` (sign handled the same way the existing theta line already does: `${v>=0?'+':''}${...}`) |
| Θ (theta) | ≤1dp, `.toFixed(1)` | Already compliant (`candidate.theta.toFixed(1)`) — unchanged |
| Score | Integer, `Math.round()` | Already integer per the `pickerCandidate` schema (`min(0).max(100)`); keep the `Math.round()` guard for defense |
| Breakdown captions (slope/fwdEdge vol-pts, gexFit %, debitFit $) | Already compliant — `formatBreakdownCaption()` in `CandidateCard.tsx` already rounds every branch (1dp vol-pts, integer %, whole $) — unchanged, reused verbatim in the new factor rows |
| IV (frontLeg.iv, backLeg.iv, fwdIv) | ≤2dp as a percent, `(v*100).toFixed(1)` | Already compliant where currently rendered (`TermStructureChart`, `WhyPanel`) — unchanged |

This rounding table applies identically wherever these fields render across BOTH desktop and
mobile trees (table, verdict hero, `MobileScorecard`'s checklist and context line, subline in the
detail-pane header) — one formatting rule, many call sites, never a per-screen exception.

## Paste Flow + Term Structure Polish (AUI-05, D-05)

- Chart height `H` grows from `230` to `320` (width `W` stays `760`; `aspect-[760/320]` on the
  `ChartContainer` wrapper) — gives the existing "short f"/"long b" `ReferenceDot` labels and the

- Leg `ReferenceDot` radius grows from `r={5.5}` to `r={7}`; label `fontSize` grows from `12` to

  more prominently at the larger chart size.
- Event `ReferenceLine`s (currently `opacity={0.3}`, no in-chart label — only the below-chart

  event, and `opacity` rises from `0.3` to `0.5`. This is the literal "event chips visually tied
  to the curve kinks" ask: the in-chart line now carries the same name the chip below it does, so
  the eye can trace one to the other instead of only inferring from x-position alignment.

## Live Sidecar Data — Visual Contract (AUI-07, D-07)

This phase's data-plumbing decisions (exactly which scoring inputs may read live vs. must stay
point-in-time) are RESEARCH's job, not this UI contract's — but the VISUAL vocabulary for "live
vs. stale" already exists and this phase must reuse it, not invent a new badge:

- **`LiveStatusBadge`** (`apps/web/src/components/LiveStatusBadge.tsx`, unchanged component) is
  mounted once in the desktop "Risk profile" `PanelHeading` action slot (next to Combine/Copy)
  and once in `MobileAnalyzerChart`'s chrome row — wired to `useLiveStream()` exactly as
  `Overview.tsx` already wires it (same `status`/`lastTickAt`/`isRth`/`hasReceivedFirstTick`/
  `isReconnecting`/`onReconnect` props). No new visual states, no new copy — LIVE/QUIET/STALLED
  render with the SAME tokens Overview already established.
- The T+0 payoff spot and any other live-eligible marker in the Analyzer follow the SAME
  live-aware-spot seam LIVE-04 established for Overview (`status==='live' && liveSpot!==null ? liveSpot : snapshot spot`)
  — a single seam, not a new one, per that requirement's own "one live-aware spot seam" law.
- The verdict hero's score/factor groups are NEVER recomputed on a live tick — they stay
  snapshot-derived, and their provenance is the hero footer's existing `as of {HH:MM} · {source}`
  line (see Verdict Hero Contract above) — this is what keeps "if the displayed score was
  computed on snapshot data, its as-of says so" true without inventing a second badge system.
- Quiet/stalled fallback never silently swaps a value with no visual cue: `LiveStatusBadge`
  itself IS that cue (its label changes to QUIET/STALLED) — no separate per-number stale
  styling is introduced beyond what `LiveStatusBadge` already communicates (catch #26's law:
  never a silent `?? fallback`, satisfied by the badge's own honesty, same as Overview).
