<objective>
Build the TS sidecar HTTP-client adapter behind the existing `ForFetchingChain` port and
its mandatory in-memory twin, TDD red→green over the scaffold committed in 11-01. The
adapter calls `GET {SIDECAR_URL}/sidecar/chain?root=...`, `safeParse`s the response with an
adapter-local Zod schema (D-08), and returns `Result<RawChain, FetchError>` — exactly the
`http/cboe.ts` pattern. This is the TS half of JRNL-02; 11-06 wires it into
`selectChainSource` as the new `schwabFetchChain` input.

Purpose: Re-source the chain through the sidecar without touching `packages/contracts`
(D-08) and without breaking the hexagon — the new adapter is a driven adapter implementing
an existing core port, with an in-memory twin shipped in the same PR (D-07).

Output: `makeSidecarChainAdapter` + `SidecarChainResponseSchema`; `makeMemorySidecarChainAdapter`
twin; barrel exports; the 11-01 RED scaffold turned green.
</objective>
