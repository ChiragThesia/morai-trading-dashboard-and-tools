<objective>
Add the `option_type` enum column to the `calendars` table (D-01) — the one schema change Phase 3 requires. Without it the snapshot job cannot resolve which `leg_observations` rows belong to a calendar's legs (resolution key is `(underlying, strike, option_type, expiry)`). This plan modifies the schema source, generates the migration, AND applies it to the live database. Build/typecheck pass without the push (types come from schema source, not the live DB) — so this plan must explicitly run the push or every live-DB verification downstream produces a false-positive then fails at runtime on a missing column.

Purpose: D-01 leg-identity foundation. Wave 1 alongside plan 01.
Output: Updated `schema.ts`, a new generated migration, and a verified live schema push.
</objective>
