<objective>
Build the snapshot vertical slice — the heart of the journal. The `snapshot-calendars` use-case resolves each open calendar's front and back legs from `leg_observations`, computes net marks / net greeks / term_slope / pnl_open / DTEs (D-05), handles unsolvable legs by writing `'NaN'`-stamped rows rather than skipping (D-06), and persists idempotently on the composite PK. The Phase 2 `fetchChain` is extended so every open calendar's exact legs bypass the DTE/band filter (D-04). The job is chain-triggered off `compute-bsm-greeks` success (D-03), gated by RTH+holiday, and wired into the worker. After this plan, registered calendars accrue journal rows on the 30-min RTH cadence.

Purpose: CAL-02 (snapshot row), CAL-04 (closed calendars excluded), CAL-05 (snapshot gate), D-03/D-04/D-05/D-06. This is the "snapshot" step of register→snapshot→read.
Output: snapshot use-case + tests, leg-resolution + snapshot repos + memory twins + shared contract test, fetchChain D-04 extension, snapshot handler + compute-chain trigger, worker composition wiring.
</objective>
