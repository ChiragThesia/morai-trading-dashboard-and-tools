    ForPersistingCotObservation; listCotObservations: ForReadingCotObservations }`. `insertCotObservation`
    does a Drizzle parameterized `insert(cotObservations).values({...row}).onConflictDoNothing({ target:
    [cotObservations.contractCode, cotObservations.asOf] })` — DO NOTHING (not DO UPDATE) so a re-run for
    the same week is a true no-op (D-09; a late-revised report is out of scope this phase). Map `asOf`

    `CotObservationRow` (integers as numbers, asOf as the YYYY-MM-DD string, publishedAt as Date). Wrap
    both in try/catch → `err(StorageError)`. Author the shared
    `runCotObservationsContractTests(makeRepo)` in `__contract__/cot-observations.contract.ts` including
    the explicit idempotency case (insert week → re-insert same as_of → assert exactly one row for that

    `describe.skipIf`. Export `makePostgresCotObservationsRepo` (+ type) from
