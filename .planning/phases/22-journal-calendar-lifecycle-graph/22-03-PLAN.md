<objective>
Wire the enriched series end-to-end on the server: a thin hexagon-pure use-case (`getCalendarLifecycle`)
that wraps the existing `ForReadingJournal` port and maps each row through `computeForwardVol` +
`computeAttributionSeries`, exposed via a new additive JWT-gated route `GET /api/journal/:calendarId/lifecycle`
and a parallel `get_journal_lifecycle` MCP tool. No new port, no new adapter, no migration — this phase reads
the existing `calendar_snapshots` table only.

Purpose: the web hook + chart (Plans 22-04/05) need one enriched endpoint that already carries forward vol
and per-interval attribution, so the browser stays presentational. Mounting inside `authReadGroup` is the
phase's single named security requirement (Pitfall 5).
Output: `makeGetCalendarLifecycleUseCase` / `ForRunningGetCalendarLifecycle` / `LifecycleSnapshot` in
`@morai/core`; `journalLifecycleRoutes` + `registerGetJournalLifecycleTool` in `apps/server`; composition wiring
in `main.ts`.
