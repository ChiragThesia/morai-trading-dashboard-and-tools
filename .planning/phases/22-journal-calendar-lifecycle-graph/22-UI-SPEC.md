## Attribution Idiom Decision (resolves CONTEXT.md Open Question 3)

**Resolution: keep both.** The hero panel's stacked-area-around-zero (cumulative, over time) and
the rail's waterfall bridge (entry→now, snapshot-styled) answer different questions and neither
subsumes the other:

- **Hero (center, temporal):** shows *when* the gamma-vs-theta collision happened — the whole
  point of D-01 is making the late-move bite visible in time, which only a time-series can do.
- **Rail bridge (right, snapshot):** shows the current *totals* at a glance, ledger-style, without
  requiring the user to read a chart. It reads the same values as the hero panel's rightmost
  point (or the crosshair-scrubbed point — see Interaction below), never an independent
  computation.

**Crosshair-scrub sync (new, since neither v3's script nor the research settled this):** the rail's
P&L bridge is reactive to the hero crosshair. Default (no hover): bridge shows the latest/"now"
cumulative values. While hovering the chart: bridge re-renders showing the hovered date's
cumulative values, with a small `"as of {day}"` mono label above the bridge so it's never
ambiguous which point it reflects. This is a Claude's-discretion interaction contract, not a
CONTEXT-locked decision — flag it to the planner as a design call, not a re-litigated one.

## Spacing Scale

Declared values (must be multiples of 4) — for this phase's new layout-level spacing (panel gaps,
card padding, column gaps):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/dot gaps in beat rows, chip gaps |
| sm | 8px | Card internal row spacing (bridge rows, kv rows) |
| md | 16px | Panel internal padding (matches existing `Panel` = `p-3` = 12px; round up new cards to `p-4`) |
| lg | 24px | Gap between masthead and chart card |
| xl | 32px | (not used at this phase's scale — reserved) |

**Exception (inherited, not introduced by this phase):** in-chart SVG labels, mono data cells, and
dense chrome (chip padding, pill borders) use fine-grained non-4-multiple px values (7px, 9px,
9.5px, 11px, 13px) — this is the *existing* Morai trading-terminal typographic density
established in `Journal.tsx`, `PayoffChart.tsx`, and `system/index.tsx` (e.g. `px-[9px]`,
`text-[10px]`). This phase's new chart and rail cards continue that established convention rather
than force layout-only spacing onto data-dense typography. The 8-pt scale above governs
*macro* layout (column gaps, card padding, section breaks) only.

## Layout & Chart Geometry (Claude's discretion, per CONTEXT — now locked as this phase's contract)

- **SVG viewBox:** `0 0 840 700`. `preserveAspectRatio="xMinYMin meet"` — **not** `"none"` like
  `PayoffChart.tsx`. A multi-panel stacked chart must never let its sub-region proportions skew
  independently of each other; `meet` scales the whole viewBox uniformly and never distorts one
  panel relative to another.
- **Region heights (logical units, top→bottom, matching v3's own layout ratios):**
  - Title/legend band above hero: 20
  - P&L Attribution (hero): 210
  - Vol & Term Structure: 120
  - 4× signed greek small-multiples (delta/gamma/theta/vega): 28 each + 8 gap = 144 total
  - Price vs Strike: 60
  - X-axis date-label band: 24
  - Top/bottom chart-card padding: ~40 (matches existing `chartcard` `padding: 8px 8px 2px` scaled)
- **Crosshair:** one shared vertical line (`--color-line2`, opacity 0.3–0.6 on hover) spanning
  from the top of the Attribution panel to the bottom of the Price panel — reuse
  `PayoffChart.tsx`'s exact `localPoint` → viewBox-space → logical-x mapping technique
  (`scaleX = SVG_W / svgRect.width`) rather than re-deriving it.
- **Tooltip:** one fixed HTML tooltip (not SVG), reusing `PayoffChart.tsx`'s existing positioning
  math (`crosshair.x / SVG_W * renderedWidth`, clamped to container width) and visual treatment
  (`rgba(8,11,16,.97)` background, `--color-line2` border, 8px radius, JetBrains Mono 10.5px).
  **Locked row order:** day label + DTE front/back → net P&L (bold, `--color-txt`) → theta delta
  (`--color-up`) → vega delta (`--color-blue`) → delta-gamma delta (`--color-violet`) → forward
  vol (`--color-amber`) → SPX spot (`--color-muted`). When the hovered index is a feed gap: show
  **only** `"feed lapsed — no data"` in `--color-dim`, suppressing every other row (never show
  interpolated/zero values for a gap).
- **Inline hero legend:** always renders all 4 entries — theta / vega / delta-gamma / residual —
  even when a series' value is currently ~0. The residual entry is never dropped (D-05).

## Honest Data States (D-05 — locked)

| State | Treatment |
|-------|-----------|
| Feed gap mid-series (`spot="0"` or non-finite greeks/IV) | Every panel's line breaks (new SVG subpath `M`, not `L`) at the gap — **never interpolated, never dashed across the gap**. Stacked-area fills flush/close at the gap boundary rather than bridging it (matches v3 script's `flush()` pattern). |
| Hovering a gap index | Crosshair still shows (so the user can see *where* the gap is), tooltip shows only `"feed lapsed — no data"` (`--color-dim`) |
| Attribution residual | Always rendered — its own faint band + always-present legend entry, regardless of magnitude. Never hidden, never folded into another series. |
| Calendar with 0–1 usable snapshots (too new, or pre-history entry/exit-only) | Reuse existing `PreHistoryStub` treatment (dashed border box) but with a **new** copy variant for "too new" (see Copywriting below) distinct from the existing pre-Jun-12 copy |
| Loading | Reuse existing `isPending` skeleton idiom exactly (`bg-line opacity-40`, `aria-busy="true"`) from `Journal.tsx`'s `LifecycleSection` |
| Fetch/compute error | New state — see Copywriting → Error state |
