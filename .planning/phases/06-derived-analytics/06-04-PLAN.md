<objective>
The first working analytics slice, end-to-end and thin: term-structure. It is the simplest of
the three outputs (value = `back_iv − front_iv`, already stored as `calendar_snapshots.term_slope`
— a passthrough with a strict no-drift equality), so it is the right first vertical slice. This
plan creates the `compute-analytics` job itself (chain-triggered after snapshot-calendars), wires
the term-structure write, and exposes the series via GET + MCP. After this plan a user can run a
cycle and read a real term-structure series by API and Claude Code.

MVP slice check: after 06-03 a user could not read any analytics; after THIS plan a user can run
a snapshot cycle and read `GET /api/analytics/term-structure` / MCP `get_term_structure` with
real data. That is a new user-facing capability — a true vertical slice.

Purpose: ANLY-02 (term-structure observations) + the ANLY-03/MCP-02 read surface for it, plus the
compute-analytics job that 06-05 will extend with skew. The term_slope-equality invariant is the
R3 prohibition (MUST NOT recompute independently).

Output: term-structure repo + twin + contract tests, the compute-analytics use-case (term-
structure half) + job + chain trigger, GET /api/analytics/term-structure + MCP get_term_structure.
</objective>
