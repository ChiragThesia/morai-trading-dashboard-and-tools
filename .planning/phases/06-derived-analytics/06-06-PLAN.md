<objective>
Close CR-01 and CR-02 — the cycle-resolution seam in `computeAnalytics`. Today the use-case
reads the smile by `readSmile(now())` (exact-equality on `leg_observations.time`, which is the
broker `observedAt`, never equal to a fresh `now()`) and stamps skew/RR rows with `snapshotTime =
now()`. In production this returns 0 smile rows (CR-01) and defeats `onConflictDoNothing` on
re-run (CR-02). The green suite hides this because the in-memory twin and the use-case tests reuse
ONE `now()` Date for both the seed time and the read.

Purpose: make the skew/RR half follow the same data-anchored pattern term-structure already uses
(`readSnapshotsForCycle` = latest snapshot <= now), so all three analytics tables share ONE
resolved cycle instant per cycle and re-runs are idempotent.

Output: a bounded "latest leg-observations cycle <= anchor" smile read on the leg-observations
repo + memory twin + shared contract; a `computeAnalytics` that resolves the anchor from data and
stamps all three tables with it; a Postgres testcontainer seam suite using DISTINCT timestamps that
FAILS on today's code and an idempotency property (run twice -> 0 new rows).
</objective>
