  <behavior>
    Signature: `interpolateRiskReversal(smile: ReadonlyArray<SmileQuote>): number | null`
    where each SmileQuote has a numeric `iv` and a numeric `delta` (put deltas negative, call
    deltas positive). NaN-stamped / null-delta points are filtered out before interpolation.

    Method (linear-in-delta, locked in SPEC "you decide"):
    1. Split usable points into puts (delta < 0) and calls (delta > 0).
    2. For the 25őĒ PUT: target delta = ‚ąí0.25. Find the two put points whose deltas bracket ‚ąí0.25
       (one ‚Č§ ‚ąí0.25, one ‚Č• ‚ąí0.25) and linearly interpolate IV across delta. If ‚ąí0.25 lies outside
       the available put-delta range (cannot bracket), the put IV is unavailable.
    3. For the 25őĒ CALL: target delta = +0.25, same bracketing/interpolation over call points.
    4. risk_reversal = IV(25őĒ put) ‚ąí IV(25őĒ call).
    5. If EITHER side cannot be bracketed ‚Üí return `null` (never extrapolate, never fabricate).

    WORKED NUMERIC ACCEPTANCE EXAMPLE (the example test MUST assert this within tol=1e-6):
    Smile points (delta, iv):
      put A:  delta ‚ąí0.20, iv 0.180
      put B:  delta ‚ąí0.30, iv 0.220
      call C: delta +0.30, iv 0.150
      call D: delta +0.20, iv 0.130
    25őĒ put: target ‚ąí0.25 lies between ‚ąí0.20 (A) and ‚ąí0.30 (B). Linear in delta:
      fraction = (‚ąí0.25 ‚ąí (‚ąí0.20)) / (‚ąí0.30 ‚ąí (‚ąí0.20)) = (‚ąí0.05)/(‚ąí0.10) = 0.5
      IV(25őĒ put) = 0.180 + 0.5¬∑(0.220 ‚ąí 0.180) = 0.200
    25őĒ call: target +0.25 lies between +0.20 (D) and +0.30 (C). Linear in delta:
      fraction = (0.25 ‚ąí 0.20)/(0.30 ‚ąí 0.20) = 0.5
      IV(25őĒ call) = 0.130 + 0.5¬∑(0.150 ‚ąí 0.130) = 0.140
    risk_reversal = 0.200 ‚ąí 0.140 = 0.060   ‚Üí  expect(rr).toBeCloseTo(0.06, 6)

    NULL CASE (example test): a smile whose put deltas are all shallower than ‚ąí0.25 (e.g. max
    |delta| = 0.18) cannot bracket ‚ąí0.25 ‚Üí returns null. Likewise an all-calls or all-puts smile
    returns null.

    PROPERTY TESTS (fast-check, per tdd.md numerical rule):

    - Order-independence: shuffling the smile array does not change the result.
