<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stored DB row → domain verdict | A persisted jsonb blob is untrusted on read — it must re-validate through the contract schema |
| retry / partial-failure → duplicate write | A pg-boss retry could re-attempt a cohort's inserts; idempotency must make that a safe no-op |
| dual-source chain rows → snapshot read | schwab_chain rows must not silently vanish from the exit advisor's view |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-26-07 | Tampering | exit_verdicts double-write (retry/race) | high | mitigate | onConflictDoNothing on composite PK (observed_at, calendar_id); WR-01 contract test asserts a differing second blob does not overwrite |
| T-26-08 | Tampering/DoS | corrupted stored verdict blob | medium | mitigate | Zod-parse the blob on read; a parse failure maps to StorageError, never a silently-invalid domain shape |
