**Gathered:** 2026-07-03

<domain>
## Phase Boundary

Bring the Analyzer's payoff interactivity onto the **Overview** payoff hero — future-date
projection + per-calendar series selection — styled to match Thinkorswim's Analyze → Risk
Profile chart, plus richer calendar dates in the positions box. **View-only over the real book.**

In scope: OVW-03 (positions-box dates), OVW-04 (TOS-fidelity graph), OVW-05 (date picker),
OVW-06 (per-calendar series selection).

Out of scope: simulated/example calendars, "Add Simulated Trades", candidate overlays — those
belong to the Analyzer → Picker redesign (Phases 18–19). The Overview never adds hypothetical
positions; it only shows and projects the real book.

</domain>

<decisions>
## Implementation Decisions

### Date-projection control (OVW-05)
- **D-01:** Calendar **date picker + `‹ ›` day-step arrows** (TOS "Date:" style). Picking a date
  reprojects the "today"/T+0 curve to **T+that-date** via the scenario-engine's existing
  `daysForward` parameter; the **@exp curve stays fixed**. Default = today.
- **D-01a:** Date range clamps to a sensible window (today → the earliest front expiry in the
  book; a calendar projected past its front expiry is degenerate). Exact clamp = planner's call.

### Calendar series selection (OVW-06)
- **D-02:** **Follow TOS's Risk Profile include behavior** — deselecting a calendar must update the
  **chart** (today the row checkboxes drive only the P&L total + greeks, NOT the graph — that
  decoupling is the exact gap the user hit). ⚑ **RESEARCH REQUIRED:** study how TOS's "Positions
  and Simulated Trades" checkboxes couple to the Risk Profile curve — does unchecking remove a
  calendar from the chart **and** the aggregate, or chart-only? Replicate TOS exactly.
- **D-02a:** Working assumption pending research: unified (one checkbox → chart + total + greeks,
  matching the Analyzer). Do not finalize the coupling until the TOS-behavior research lands.

### TOS graph fidelity (OVW-04)
- **D-03:** Match the TOS Risk Profile combined chart **exactly** — magenta "today/date" curve +
  cyan "@exp" curve, TOS-style even x-axis ticks and auto-scaling (so the @exp curve no longer
  dwarfs a near-flat T+0). ⚠ This **overrides** Morai's existing violet/gray palette + design
  system for the payoff hero — **explicit user instruction** (user instruction > design system).
  Expect a UI-checker flag on brand divergence; it is pre-approved for this hero only.
- **D-03a:** ⚑ **RESEARCH REQUIRED:** capture TOS's exact curve colors, gridline/tick style, axis
  scaling, and price-slice/date label conventions from the reference screenshots.

### Positions box (OVW-03)
- **D-04:** Replace the compact "DTE" cell with **expiries + calendar width, DTEs inline**:
  `Aug 8 → Sep 5` (front → back expiration dates, short-month) with `32d/59d · 27d wide` beneath
  (front DTE / back DTE · days-between). Days-between = `backDte − frontDte` (the calendar width).

### Claude's Discretion
- Date-picker implementation: prefer a **zero-dep** control (native `<input type="date">` or a
  small custom popover over existing primitives) — the v1.2 "zero new deps" lock holds. Step-arrow
  increment = 1 day. Exact TOS hex colors, tick density, and axis-scaling algorithm — planner /
  researcher decide within D-01..D-04.

</decisions>

## Existing Code Insights

### Reusable Assets
- **Analyzer's `daysForward` slider + include-checkbox pattern** (`Analyzer.tsx`) — direct template
  for OVW-05/OVW-06; port a trimmed, view-only version (no simulated trades).
- **`scenario-engine` `daysForward`** — already parameterized end-to-end (`calendarNetPrice` /
  `bookPL` / `bookPLAtExpiry` all take `daysForward`); OVW-05 just needs to feed a non-zero value.
- **PayoffChart props from Phase 17** (`highlightedPositionId`, curve props, `excludedFromT0Count`)
  — the restyle (OVW-04) reworks the curve/axis rendering, not the prop contract.

### Established Patterns
- Overview positions table tracks **exclusions** via an `excluded` Set (not inclusions);
  `included = !excluded.has(key)`. This set drives the P&L Net total + live greeks.
- **The graph is decoupled today:** `calendarPositions` for `repriceScenario` are built with
  `included: true` hard-coded (~`Overview.tsx:144`), so the row checkboxes do NOT affect the payoff
  curve. Wiring that link is the core of OVW-06.
- Overview is **view-only** (`view-only · Analyzer →`) — do not add editing/simulation.

### Integration Points
- OVW-06: route the row `excluded` set → the graph's `calendarPositions` inclusion.
- OVW-05: date-picker state → `params.daysForward` into `repriceScenario`.
- OVW-04: restyle `PayoffChart` curve/axis rendering to TOS.
- OVW-03: reformat the positions-table DTE cell.

### Constraints
- **Zero new dependencies** (v1.2 lock) — date picker must be native/existing primitives.
- TDD red→green, no `any`/`as`/`!`, `Result<T,E>` (project CLAUDE.md rules).
