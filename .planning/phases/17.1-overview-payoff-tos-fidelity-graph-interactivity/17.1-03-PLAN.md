<objective>
Fix OVW-06: lift the positions-table `excluded` Set out of `PositionsTable` up to `Overview`
so a single source of truth drives BOTH the payoff chart's per-calendar inclusion AND the
table total/greeks — replicating TOS's unified Risk Profile include behavior (D-02/D-02a).
Extend `positionSetSignature` to encode inclusion so the y-domain recomputes on toggle
(RESEARCH Pitfall 2), and prove the CR-01 non-convergence exclusion is not regressed.

Purpose: today `calendarPositions` are built with `included: true` hardcoded
(`Overview.tsx:144`), so the row checkboxes reach only the P&L total, never the graph. That
decoupling is the literal bug this requirement exists to close. The fix is state-lifting, not
a second synchronized Set (RESEARCH Pitfall 3 / anti-pattern).
Output: single lifted inclusion state wired through the calendar build, the signature, and a
controlled `PositionsTable`.
</objective>
