## Artifacts this phase produces

New file `apps/web/src/lib/date-projection.ts` exporting these signatures (names are the
contract Plan 04 depends on):

- `toDateInputValue(d: Date): string` — local `YYYY-MM-DD` string for a Date (input value / min / max attr).
- `parseLocalDateInput(value: string): Date | null` — parse `YYYY-MM-DD` as a **local** midnight Date via `new Date(y, m-1, d)`; return `null` for empty/malformed (no `Invalid Date`).
- `daysBetween(from: Date, to: Date): number` — whole calendar days between two dates, computed at local midnight.
- `resolveDaysForward(value: string, today: Date, maxDaysForward: number): number` — the single entry point the picker calls: parse → `null` ⇒ `0`; else `clamp(daysBetween(today, picked), 0, maxDaysForward)`.
- `computeProjectionBounds(includedFrontDtes: ReadonlyArray<number>, today: Date): { readonly minIso: string; readonly maxIso: string; readonly maxDaysForward: number }` — `maxDaysForward = includedFrontDtes.length > 0 ? Math.min(...includedFrontDtes) : 0`; `minIso = toDateInputValue(today)`; `maxIso = toDateInputValue(today + maxDaysForward days)`.

New file `apps/web/src/lib/date-projection.test.ts`.
