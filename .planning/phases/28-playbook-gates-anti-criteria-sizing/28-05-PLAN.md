<objective>
Add the event-calendar bucket (PLAY-04): a second candidate universe for short-gap (3-10 day)
calendars that intentionally own a tier-1 event between their legs, scored with an
event-appropriate rule set.

This is a thin wrapper, not a second engine. selectCandidates already computes backEvents per
candidate; selectEventCandidates calls it with the back-leg gap window parameterized to [3,10]
days (making the existing BACK_DTE_MIN_GAP/MAX_GAP defaults optional params — a small additive
change), then post-filters to candidates whose back leg owns an event (backEvents.length > 0).
These are scored with a SEPARATE EVENT_RULE_SET_METADATA that promotes the currently-experimental
backEventBonus to an active weight with its own sum-100 invariant — the primary registry and its
locked weight-sum test are never touched. Results carry a bucket tag for distinct Analyzer
rendering.

Purpose: PLAY-04 — score event-owning short-gap calendars through their own rule set.
Output: selectEventCandidates + EVENT_RULE_SET_METADATA + bucket-scored, bucket-tagged snapshot
candidates.
</objective>
