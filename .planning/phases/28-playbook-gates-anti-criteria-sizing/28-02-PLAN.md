Deliver the anti-criteria brakes (PLAY-02): the one genuinely-new piece of infrastructure this
phase needs — a bulk "recently-closed calendars since date X" read — plus the pure brake
evaluators that turn its rows (and the existing open-calendar count) into the two boolean
inputs Plan 01's resolveEntryGate already accepts.

same PR (architecture rules 8 + tdd.md "new port → in-memory + contract test, same PR").

Purpose: Pause new entries on the user's own risk rules — max open (6) and a realized-loss
cooldown (−25% → 2 business days) — per USER DECISION 2.

</objective>
