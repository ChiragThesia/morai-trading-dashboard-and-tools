<objective>
Establish the market-level entry gate as a pure, tested domain module plus the docs that
describe the whole playbook-port design up front (docs-before-code, per workflow.md).

`resolveEntryGate` is the single cohort-level function that decides whether the picker may
emit new candidates: it bands VIX and the VIX/VIX3M ratio into open/penalty/blocked with
hysteresis, fails CLOSED to a loud GATE BLIND state when the macro data is stale (USER
DECISION 1), and accepts already-resolved brake booleans (fed by Plan 02's port) so both
anti-criteria brakes route through this same one call.

This plan ships ZERO wiring — the gate is not yet called by the use-case (that is Plan 03).
It is the pure-function foundation every later plan builds on.

Purpose: A market-regime pre-condition that never became a per-candidate filter (the retired
`term-inversion` scar) and never silently degrades to "open" on missing data.
Output: entry-gate.ts (+ tests), and the docs/architecture/playbook-gates.md contract.
</objective>
