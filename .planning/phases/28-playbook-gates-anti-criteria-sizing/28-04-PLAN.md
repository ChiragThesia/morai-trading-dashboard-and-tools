<objective>
Add VIX-tiered discrete sizing (PLAY-03) and, time-box permitting, the additive
autoTuneTargetDelta band preference (PLAY-05).

Sizing is a named-constant registry in picker/domain/sizing.ts (rules.ts RULE_SET_METADATA
style) mapping the SAME VIX ladder the gate uses to a user-set contract count per tier — never
a derived optimum. The engine resolves the tier from the cohort VIX and ships it on the
snapshot; the Analyzer renders it (Plan 06). The proposed default counts (2/2/1/0) and tier
edges are flagged for UAT confirmation.

autoTuneTargetDelta is the milestone's most-optional requirement (explicit time-box: ship a
minimal additive tilt or defer first). The evidence is directional-only, so this plan ships the
smallest possible version — a VIX-tuned nudge of the band-scan target-delta toward the further-
OTM edge as VIX rises, clamped inside the existing delta band — OR, if it fights the existing
slope/fwdEdge signals in testing, documents the defer in playbook-gates.md and drops it (the
milestone allows dropping PLAY-05 first).

Purpose: PLAY-03 discrete sizing + PLAY-05 additive delta preference.
Output: sizing.ts registry + additive sizing snapshot field; a minimal autotune tilt or a
documented defer.
</objective>
