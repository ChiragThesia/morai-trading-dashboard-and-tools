<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /api/stream/ticket | Supabase JWT verified at the edge (mint gate) |
| browser → GET /api/stream | Header-less EventSource authed by opaque ticket only |
| server → sidecar | Internal fetch of /sidecar/events + /sidecar/positions (private net) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-05-01 | Spoofing | GET /api/stream auth | mitigate | Mounted OUTSIDE authReadGroup but requires a valid single-use ticket via redeemTicket; missing/invalid/expired/used → 401 before any SSE (Pitfall 7) |
| T-12-05-02 | Information Disclosure | JWT in URL | mitigate | EventSource carries only the opaque ticket in the query param; the JWT never leaves the Authorization header on the POST mint (D-01 — no JWT in logs/history) |
| T-12-05-03 | Elevation of Privilege | unauthenticated mint | mitigate | POST /api/stream/ticket sits inside the Supabase JWT authReadGroup; only authenticated users can mint |
| T-12-05-04 | Tampering | malformed sidecar frames | mitigate | Every frame Zod safeParsed in sidecar-sse.ts; unparseable frames dropped (no any/as/!) |
| T-12-05-05 | Denial of Service | ticket-mint flooding | mitigate | Single-use 30s tickets bound per userId (12-04); a connecting browser holds at most a small number of live tickets; reconnect/backoff in the consumer prevents tight loops |
| T-12-SC | Tampering | npm installs | accept | Zero new packages — hono/streaming, jose, crypto.randomUUID already present (RESEARCH audit) |
</threat_model>
