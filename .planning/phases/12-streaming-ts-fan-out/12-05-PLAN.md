<objective>
Stand up the server side of the fan-out: the opaque-ticket-authed SSE endpoint (D-01,
STRM-03), the sidecar SSE consumer that parses each frame, BSM-recomputes greeks (D-02), and
feeds the coalescer, the real ForReconcilingPositions adapter over /sidecar/positions
(STRM-05), the JWT-gated ad-hoc subscribe proxy (POST /api/stream/subscribe → sidecar, D-05/SC6),
and the composition-root wiring — including adding the missing SIDECAR_URL to the server config.
POST /api/stream/ticket + POST /api/stream/subscribe are JWT-guarded; GET /api/stream is
ticket-guarded and sends a reconcile event first (STRM-05) so a connecting/reconnecting browser
never shows a gap.

Purpose: STRM-03 (authed N-client fan-out, JWT verified at the edge via the mint route; header-
less EventSource via ticket) + STRM-05 (reconcile-on-connect) + STRM-01 expanded (ad-hoc
subscribe relayed to the sidecar so any OCC symbol the user enters goes live — D-05, SC6).
Output: stream.routes.ts (ticket + subscribe + SSE), sidecar-sse.ts, sidecar positions-reconciler,
config + main.ts wiring.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /api/stream/ticket | Supabase JWT verified at the edge (mint gate) |
| browser → POST /api/stream/subscribe | Supabase JWT verified at the edge; authed mutation of the shared stream subscription set |
| browser → GET /api/stream | Header-less EventSource authed by opaque ticket only |
| server → sidecar | Internal fetch of /sidecar/events + /sidecar/positions + /sidecar/subscribe (private net) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-05-01 | Spoofing | GET /api/stream auth | mitigate | Mounted OUTSIDE authReadGroup but requires a valid single-use ticket via redeemTicket; missing/invalid/expired/used → 401 before any SSE (Pitfall 7) |
| T-12-05-02 | Information Disclosure | JWT in URL | mitigate | EventSource carries only the opaque ticket in the query param; the JWT never leaves the Authorization header on the POST mint (D-01 — no JWT in logs/history) |
| T-12-05-03 | Elevation of Privilege | unauthenticated mint | mitigate | POST /api/stream/ticket sits inside the Supabase JWT authReadGroup; only authenticated users can mint |
| T-12-05-04 | Tampering | malformed sidecar frames | mitigate | Every frame Zod safeParsed in sidecar-sse.ts; unparseable frames dropped (no any/as/!) |
| T-12-05-05 | Denial of Service | ticket-mint flooding | mitigate | Single-use 30s tickets bound per userId (12-04); a connecting browser holds at most a small number of live tickets; reconnect/backoff in the consumer prevents tight loops |
| T-12-05-06 | Elevation of Privilege / DoS | POST /api/stream/subscribe — authed mutation of the shared stream | mitigate | Mounted INSIDE authReadGroup (only authenticated users); the stream is a single shared session so any user's ad-hoc symbol affects all viewers — bounded by the sidecar SubscriptionManager 490-cap LRU (12-02), and the OCC parse rejects junk before it reaches the broker. Per-user rate-limiting is out of scope this phase (single-operator deployment) — documented residual |
| T-12-SC | Tampering | npm installs | accept | Zero new packages — hono/streaming, jose, crypto.randomUUID already present (RESEARCH audit) |
</threat_model>
