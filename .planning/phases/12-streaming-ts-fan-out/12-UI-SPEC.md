**Inherited from Phase 9 — no changes.** Reproduced here as the binding reference.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline element gaps |
| sm | 8px | Table cell padding, live-value cell padding |
| md | 12px | Card internal padding (Tailwind `p-3`), grid gap |
| lg | 16px | Screen container horizontal padding |
| xl | 24px | Row gap between Row 1 and Row 2 |
| 2xl | 32px | Not used in Phase 12 surfaces |
| 3xl | 48px | Not used in Phase 12 surfaces |

**Spacing Exception — 12px `md` token (inherited from Phase 9):**
The `md` token maps to 12px (Tailwind `p-3`), which is NOT in the standard 8-point grid set {4, 8, 16, 24, 32, 48, 64}. This value is the shadcn/ui base-nova `card` component's default internal padding. Overriding it would require fighting the component's internal CSS and would create maintainability debt across all cards already shipped to morai.wtf. Exception approved — 12px is inherited from the Phase 9 approved design system (Phase 9 UI-SPEC, approved). Phase 12 introduces no additional non-grid spacing values.

**Touch targets:** minimum 44×44px on the "Stream Greeks" button and the ad-hoc clear (×) button.

**Typography Exception — three weights active (400, 600, 700):**
The standard contract maximum is two weights. Phase 12 uses three: 400 (body mono values and label mono), 600 (label Space Grotesk headings), and 700 (subhead KPI emphasis). Weight 700 is NOT introduced by Phase 12 — it is inherited directly from the approved Phase 9 contract (Phase 9 UI-SPEC, approved) where `subhead`-level cells (Mark, Unreal KPI values) carry weight 700 for numeric emphasis. The `display` token (24px / 700) is present in the type scale but is NOT used in any Phase 12 new surface; weight 700 enters Phase 12 exclusively through the `subhead` token. Phase 12 introduces no new font weights beyond those locked in Phase 9.

**Typographic rules (inherited, applied to Phase 12 cells):**
- `font-variant-numeric: tabular-nums` on every live greek value cell — value widths must not shift during updates (already set globally on `body`).
- Live value cells: `font-family: var(--font-mono)` — same as existing greeks table.
- Signed values: prefix "+" for positive, "−" (U+2212) for negative. Text color confirms sign.
- Status text ("LIVE", "STALE", "RECONNECTING"): `label` token, uppercase, tracking 0.9px.

**Semantic color rules (unchanged from Phase 9):**
- `#26a69a` teal: profit/gain/live/connected — "LIVE" dot uses this because "connected" is a positive operational state.
- `#ef5350` coral: loss/risk/destructive/validation error — OCC format errors use this.
- `#f0b429` amber: warning/degraded — "STALE" and "RECONNECTING" use this because the stream is degraded, not dead.
- `#5b9cf6` blue: spot price only — not repurposed for Phase 12.

**Stale value dimming:** Do NOT use CSS `opacity` on stale cells (it creates accessibility issues and can blur against the card gradient). Instead, change `color` to `--color-dim` (#566273) on stale cells to signal the value is frozen without making it invisible. Restore to `--color-txt` on reconcile.

## Phase 12 Surface Specifications

### Surface 1 — Live Greeks Overlay (D-06, D-07)

**What it is:** The existing per-leg greeks table in the "Position" card (span 4, Row 1) and
the KPI grid above it gain live values from the EventSource stream. Values update at ~1/sec
(coalesced by the server per D-07). No layout change — this is a value overlay, not a new panel.

**Live vs static distinction:**
- A connection-status indicator (see Surface 3) appears in the "Position" CardHeading area,
  replacing the static "per spread" badge when a live stream is active.
- When live: CardHeading badge reads "LIVE" in `--color-up` (#26a69a) with a 6px pulsing dot.
- When polling-only (no EventSource): badge reads "POLL" in `--color-muted` (#7b8696) — no dot.

**Per-cell update animation (on value change, every ~1sec):**
- Target cells: Mark, Δ, Γ, Θ/d, Vega, IV columns in the per-leg greeks table; "Mark" and "Unreal" KPI cells.
- On change: set `background-color` to `--color-violetd` (#241d40) instantaneously.
- Fade: `transition: background-color 300ms ease-out` back to `transparent`.
- No directional flash (no green-on-up / red-on-down background). The text color (`--color-up` / `--color-down`) already carries directional meaning. The flash is a neutral "this value just changed" signal only.
- Frequency: at most once per second per cell (server coalesces — client applies the animation on every received tick).
- Do NOT use `@keyframes` pulse or strobe on value cells. A single 300ms fade is the maximum animation.

**Tabular-nums enforcement:** all live value cells inherit the global `tabular-nums` setting. No layout shift when digits change. Values always right-aligned in the greeks table.

**IV display:** shows BSM-computed IV (from D-02), formatted as "18.4%" (one decimal, `%` suffix). When no live data: shows the static fallback ("—" or the computed DEFAULT_IV).

**Mark display:** shows streamed mark (bid/ask midpoint fallback per RESEARCH Pitfall 4), formatted as "$47.20" or "—" when absent.

**Greeks display:** shows BSM-recomputed greeks from `@morai/quant` (D-02), formatted with `fmtGreek()` (4 decimal places, sign prefix). Same format as the existing static greeks table. No separate "live" format.

**Empty / no-data cell:** when a live tick has not arrived for a symbol within 5 seconds of stream connect, keep showing the last polling-derived value with no flash. The stale treatment (Surface 2) kicks in only on stream disconnect, not on a per-symbol tick gap.

### Surface 2 — Stale-Data Treatment (D-04)

**What it is:** When the EventSource drops (network loss, server restart, tab backgrounded),
the last received values freeze visibly — the user sees they are stale, not silently wrong.

**Stale visual treatment (applied on stream disconnect):**
- All live-sourced cells in the greeks table and KPI grid: change `color` from `--color-txt` to `--color-dim` (#566273). No CSS `opacity`.
- Transition: `transition: color 500ms ease` — a slow drift to dim (not an instant switch) signals a gradual stale condition.
- Do NOT add a strikethrough, blur, or overlay — the numbers must remain readable.
- The "Position" CardHeading badge switches from "LIVE" (teal dot) to "STALE" (amber badge, no dot). See Surface 3 for badge specs.
- The ad-hoc result row (if any) receives the same dim treatment.

**Reconnecting state (between disconnect and successful reconcile):**
- "STALE" badge changes to "RECONNECTING" badge (amber, muted text, no dot).
- Cell colors remain `--color-dim` — do not restore until the reconcile SSE event lands.
- No spinner or progress indicator in the cells themselves. The badge alone signals reconnect in progress.

**Recovery animation (on reconcile event received):**
- Restore all cell colors: `transition: color 300ms ease` from `--color-dim` back to `--color-txt`.
- Apply the value-change flash (`--color-violetd` → transparent, 300ms) on cells whose values changed in the reconcile diff.
- Badge switches from "RECONNECTING" / "STALE" to "LIVE" (teal dot).

**Stale age display:** not shown inline. A `Tooltip` on the "STALE" or "RECONNECTING" badge reveals the timestamp of the last received tick in the format "Last update: 14:32:07" (HH:mm:ss, 12px mono). Tooltip triggers on hover/focus. This avoids cluttering the card with a live clock.

### Surface 3 — Connection Status Affordance

**What it is:** A minimal, always-visible indicator of the EventSource stream state. Single
source of truth — appears in one place only: the CardHeading badge slot of the "Position" card
(top-right of the span-4 card, Row 1).

**States:**

| State | Trigger | Badge text | Dot | Colors |
|-------|---------|-----------|-----|--------|
| LIVE | EventSource connected, ticks arriving | "LIVE" | 6px teal pulsing dot | Text: `--color-up` (#26a69a); dot: same |
| STALE | EventSource dropped, no reconnect yet | "STALE" | None | Text: `--color-amber` (#f0b429); background: `--color-raise` |
| RECONNECTING | EventSource dropped, reconnect in progress | "RECONNECTING" | None | Text: `--color-muted` (#7b8696); background: `--color-raise` |
| POLL | No EventSource (SSE not available or not yet connected) | "POLL" | None | Text: `--color-dim` (#566273); no background |

**Live dot animation:** `@keyframes pulse` — scale 1.0 → 1.4 → 1.0 at 2s interval, `ease-in-out`. Apply only when state is LIVE. Stop animation on STALE or RECONNECTING.

**Badge rendering:**
- Use the existing `<Badge variant="outline">` from `components/ui/badge.tsx` with an inline-start icon slot for the dot.
- Font: `label` token — 10px, JetBrains Mono, uppercase, tracking 0.9px.
- The badge replaces the existing static `badge` prop value in `<CardHeading>` — no layout change.

**Tooltip on badge:** on hover/focus, show a `<Tooltip>` with the last-tick timestamp: "Last update: HH:mm:ss" (12px mono, `--color-muted`). If no tick has ever arrived: "No data received yet."

**Primary visual focal point:** The connection-status badge dot in the Position CardHeading is the primary visual focal point for stream state — it is always visible regardless of which position is selected and provides an at-a-glance stream health signal without interrupting the data layout.

**No global header element:** the connection status is local to the Positions screen. It does not appear in the sticky Shell header or the market strip. The status is position-data specific, not app-wide.

### Surface 4 — Ad-Hoc Instrument Picker (D-05)

**What it is:** A control to enter any OCC option symbol (e.g., a prospective SPX calendar leg)
and stream its live BSM greeks without that position being owned. Embedded in the existing
"Open" positions list card (span 3, Row 1) as a footer section, below a `<Separator>`.

**Placement:** below the owned positions list, separated by `<Separator>` with `--color-line`
(1px). No layout change to the 12-column grid. If the positions list is empty, the separator
and picker appear immediately (the picker is always accessible, not conditional on having positions).

**Input affordance:**
- Section label: `<CardHeading text="Ad-hoc lookup" />` — same style as existing CardHeadings (10px, uppercase, `--color-muted`). No badge.
- Input: `<Input>` from `components/ui/input.tsx` (base-ui `InputPrimitive` wrapper).
  - Placeholder: `SPX   260620C05000000`
  - Font: 10px JetBrains Mono on placeholder; 12px (body token) on entered text.
  - Width: 100% of the card interior (spans the full span-3 card width).
  - Height: 32px (h-8 from the Input component default).
  - Border color: `--color-line2` (#27313f) at rest; `--color-violet` (#a78bfa) on focus (focus ring — existing `ring` token).
  - Enter key submits. No separate submit button needed if the symbol is valid; the Enter key is the primary affordance.
- Submit button: `<Button variant="secondary" size="sm">Stream Greeks</Button>` — appears to the right of the input as a `flex` row. 44px minimum height (touch target). Label: "Stream Greeks" (verb + noun).
- Clear button: `×` (lucide `X` icon, 14px) — appears inside the input as a trailing icon when a symbol is entered. Clicking clears the input and removes the ad-hoc subscription. Accessible label: `aria-label="Clear ad-hoc symbol"`. Color: `--color-muted` (#7b8696) at rest; `--color-txt` (#d6dbe4) on hover.

**Symbol validation:**
- Validation runs on Enter/submit, not on keystroke (avoids error-on-typing UX).
- Use `parseOccSymbol` from `@morai/shared` as the client-side validator (already imported in Positions.tsx).
- Valid OCC: proceed to subscribe.
- Invalid OCC: show an error message below the input. Error copy: "Invalid OCC format — use 21-char Schwab format (e.g. SPX   260620C05000000)". Color: `--color-down` (#ef5350). Font: `body` token (12px mono). No red border on the input itself — the error text below is sufficient.
- Empty submit: no error shown; do nothing.

**Ad-hoc result display:**
- The subscribed ad-hoc symbol appears as a row in the positions list, below the owned positions and the `<Separator>`.
- Row appearance: same button/row style as owned positions (`background: transparent`, border on hover/selected), with one difference — a small "AD HOC" badge to its right.
  - "AD HOC" badge: 10px JetBrains Mono uppercase; background: `--color-raise` (#161d2b); text: `--color-dim` (#566273); border: `--color-line` (#1b2433); border-radius: 3px; padding: 1px 5px.
- Ad-hoc row label: `legLabel(occSymbol)` — same formatter as owned positions (e.g. "5000C 06/20/26").
- Ad-hoc row sub-label: DTE value only (no P&L — there is no position). Format: "DTE: 47" in `--color-dim`.
- Selecting an ad-hoc row: the "Position" card (span 4) shows live BSM greeks from the stream. The KPI grid shows: Mark (live from stream), Δ/Γ/Θ/Vega/IV (BSM-recomputed). Debit and Unreal KPIs show "—" (no position basis). The attribution waterfall shows an empty state (see copywriting).
- Only one ad-hoc symbol can be active at a time in this panel. Submitting a new OCC symbol replaces the previous ad-hoc entry.

**How ad-hoc differs from owned positions (visual distinction summary):**

| Property | Owned position | Ad-hoc symbol |
|----------|----------------|---------------|
| Row badge | none | "AD HOC" (`--color-dim` on `--color-raise`) |
| Sub-label | DTE + P&L | DTE only |
| KPI: Debit | dollar value | "—" |
| KPI: Unreal | dollar value | "—" |
| Attribution waterfall | populated | empty state |
| Stale dimming | yes (owned) | yes (same treatment) |
| Persistent across reload | yes (from API) | no (session only) |

## Interaction States

### EventSource lifecycle → UI state machine

```
DISCONNECTED (initial)
  → connect EventSource
  → CONNECTING (show "POLL" badge, no live values)
  → on first tick: LIVE (show "LIVE" teal dot, animate values)
  → on disconnect: STALE (dim cells, amber "STALE" badge)
    → on reconnect attempt: RECONNECTING (amber, muted text)
    → on reconcile SSE event: LIVE (restore cells, green dot)
    → on repeated failure (>60s): STALE (stay dimmed, keep badge)
```

### Value update animation timeline

```
t=0     tick arrives via EventSource → new value computed
t=0ms   cell background set to --color-violetd (#241d40)
t=0ms   cell text content updated to new value
t=300ms background transitions back to transparent (ease-out)
t=300ms animation complete — cell is visually stable
```

### Stale transition timeline

```
t=0     EventSource disconnect detected (onError / stream.close)
t=500ms cell colors transition from --color-txt to --color-dim (ease)
t=0ms   badge switches to "STALE" (amber, no dot)
t=..    browser begins reconnect (EventSource auto-retry or hook logic)
t=..    badge changes to "RECONNECTING"
t=..    reconcile SSE event arrives
t=300ms cells transition back to --color-txt
t=0ms   badge changes to "LIVE" with pulsing dot
```

### Ad-hoc picker states

| State | Visible elements |
|-------|-----------------|
| Initial (no symbol entered) | Input (empty), "Stream Greeks" button (disabled or grayed), empty-state copy below |
| Symbol entered, not submitted | Input (with text), "Stream Greeks" button (enabled), no error |
| Submit: valid symbol | Input (readonly or cleared), ad-hoc row appears below separator, live values start streaming |
| Submit: invalid OCC | Input (text preserved), error message below in `--color-down`, no row added |
| Symbol evicted (cap reached) | Transient notice replaces error slot: "Oldest ad-hoc symbol replaced." in `--color-muted`, 10px mono; disappears after 3s |
| Ad-hoc row selected | Same as owned position selected: `--color-violetd` background, `--color-violet` border, Position card shows ad-hoc greeks |
| Stream drops while ad-hoc active | Ad-hoc row dims same as owned positions (Surface 2 treatment) |
