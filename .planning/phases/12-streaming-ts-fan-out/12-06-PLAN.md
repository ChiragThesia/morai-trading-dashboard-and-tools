<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/stream/ticket + /api/stream | Client mints a ticket (Supabase-authed) then connects EventSource |
| stream frame → DOM | Parsed tick data rendered; ad-hoc OCC input is user-supplied |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-06-01 | Tampering | EventSource frame → render | mitigate | Every frame Zod-parsed (streamLiveGreekEvent/Reconcile); malformed frames dropped, never cast into the DOM |
| T-12-06-02 | Tampering | ad-hoc OCC input | mitigate | parseOccSymbol validates the 21-char format before subscribeAdHoc; invalid → inline error, no POST. A valid symbol is sent only to the Supabase-authed POST /api/stream/subscribe (apiFetch), which re-validates server-side (12-05) — client validation is UX, the server is authoritative (UI-SPEC Surface 4 + RESEARCH V5) |
| T-12-06-03 | Information Disclosure | ticket in URL | accept | Ticket is opaque + single-use + 30s TTL (12-04/05); browser-history exposure is low-value and expires fast (D-01 rationale) |
| T-12-06-04 | Spoofing | unauthed stream access | mitigate | Hook mints via apiFetch (Supabase JWT); an unauthenticated client cannot obtain a ticket so EventSource connect 401s |
| T-12-SC | Tampering | npm installs | accept | Zero new packages — all six shadcn components already installed (UI-SPEC Registry Safety) |
</threat_model>
