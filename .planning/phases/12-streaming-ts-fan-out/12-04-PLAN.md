Build the two in-memory primitives the server SSE endpoint stands on: the opaque single-use
ticket store (D-01 — mint behind the Supabase JWT, redeem at the header-less EventSource
connect) and the fan-out hub (a Set of SSE clients + a per-symbol coalescer flushing ~1/sec,
D-07). Both are pure, framework-light units with deterministic tests. This plan also lands the
automated STRM-04 regression gate proving a streaming-only session writes nothing to Postgres.

Purpose: STRM-03 auth core (ticket lifecycle: TTL, single-use, replay rejection) and STRM-04
display-only guarantee, both fully unit-tested before the route wiring in 12-05.
Output: ticket-store.ts, stream-fan-out.ts, and the STRM-04 regression test.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → server (ticket) | A ticket string arrives at the header-less EventSource connect; must be unforgeable + single-use |
| internal tick → fan-out Set | Ticks broadcast to all connected clients; dead clients must not accumulate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-04-01 | Spoofing | redeemTicket | mitigate | Tickets are crypto.randomUUID (unguessable) + single-use + 30s TTL; replay/reuse returns null |
| T-12-04-02 | Information Disclosure | TicketRecord contents | mitigate | Record holds only {userId, exp, used} — no JWT/secret/claim; ticket is opaque (D-01) |
| T-12-04-03 | Denial of Service | fan-out Set growth | mitigate | Dead clients removed on aborted + writeSSE-rejection (Pitfall 6); coalescer bounds per-symbol to one tick (D-07) |
| T-12-04-04 | Tampering | hidden Postgres write in stream path | mitigate | STRM-04 regression test fails the build if any streaming path inserts a leg_observations row |
| T-12-SC | Tampering | npm installs | accept | Zero new packages (RESEARCH audit) — crypto.randomUUID + hono/streaming already present |
