<objective>
Close the two diagnosed UAT gaps for Phase 12 (tests 2 and 3): live SSE greeks never reach the
rendered screen, and no connection badge is visible. Root cause is conclusive and is a UI-WIRING
REGRESSION only — the streaming backend (sidecar → server fan-out), the `useLiveStream` hook, and
the `LiveStatusBadge` / `AdHocPicker` components are all correct and already shipped (12-06). The
5→3 screen redesign orphaned the overlay: `App.tsx` mounts Overview/Analyzer/Journal, but the live
overlay was wired into `Positions.tsx`, which is now dead code. The mounted `Overview.tsx` renders
its OWN `PositionsTable` fed by `usePositions()` (30s REST poll) + static `computePositionGreeks`
and never imports `useLiveStream` or `LiveStatusBadge` — so greeks only change on the 30s poll or a
manual refresh (the user's report: "why is the UI only refreshing when I refresh the page?").

This plan ports the live overlay into the mounted Overview surface, surfaces the connection badge,
and removes the orphaned screen. Do NOT replan the streaming pipeline.

Dispositions decided in this plan (both stated explicitly per the gap brief):
  1. Dead-screen duplication — DELETE `Positions.tsx` + its test (it is orphaned; the redesign
     superseded it). The genuinely shared live-overlay math is extracted into a tested pure helper
     (`lib/live-position-greeks.ts`) consumed by Overview's per-row AND Net-total renders, so there
     is no divergent second table to drift.
  2. AdHocPicker (Surface 4 / SC6) — KEEP on Analyzer, where it is already wired and functional. Do
     NOT add it to Overview (a second subscription path exceeds the wiring-fix scope). SC6 is
     re-verified via Analyzer.

Purpose: Restore the live-streaming experience on the screen the user actually sees, with the
connection state visible, and eliminate the dead duplicate.
Output: a tested live-overlay helper, an Overview that overlays live greeks + shows the badge, and
the orphaned Positions screen removed — full web suite green.
</objective>

    liveTs: string | null. Types: legs is ReadonlyArray&lt;BrokerPositionResponse&gt; and liveGreeks is

    a tick causes NO scale jump. Critical scaling fact: computePositionGreeks(leg).greeks.* already
    returns kernel-per-share × netQty, and Overview then multiplies that by nq = netQty × 100. The
    live tick's bsmDelta/bsmGamma/bsmTheta/bsmVega are RAW per-share values (the same layer as the
    kernel-per-share). Therefore substitute at the per-share layer and apply the SAME downstream
    factors. Per leg, with netQty = leg.longQty − leg.shortQty and nq = netQty × 100, look up

      - Net val contribution: when tick present, tick.mark × netQty × 100 (signed by netQty); when
        absent, leg.marketValue ?? 0 (identical to netValue today).
      - Unreal contribution: when tick present, (tick.mark − leg.averagePrice) × netQty × 100; when
        absent, leg.marketValue − leg.averagePrice × netQty × 100. If ANY leg has averagePrice === null
        and no tick supplies a basis, the row's unreal is null (matches netUnreal today).
      - liveTs: the lexicographically greatest tick.ts among legs that had a tick (ISO-8601 UTC strings
