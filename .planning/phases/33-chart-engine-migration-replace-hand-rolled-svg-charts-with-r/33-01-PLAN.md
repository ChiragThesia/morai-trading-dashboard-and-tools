<objective>
Lay the foundation every other Phase 33 plan builds on: install Recharts (pinned), scaffold the
shadcn chart primitive, reconcile the architecture doc that currently records Recharts as rejected,
add the shared jsdom test harness, and empirically prove the one load-bearing library assumption
(Assumption A1: ComposedChart sibling z-order follows JSX source order in 3.9.2) before any chart is
built on it.

Purpose: The migration's whole premise is that Recharts owns scales/clipping/hover natively. That
premise rests on (a) the shadcn `ChartContainer` plumbing wiring the MORAI palette through CSS vars,
(b) a jsdom mock so `ResponsiveContainer` does not render 0x0, and (c) JSX-order z-control. This plan
delivers all three and de-risks (c) with a cheap spike, so plan 33-06 does not build 9 layers on an
unverified assumption (RESEARCH Pitfall 4 / Assumption A1).

Output: `recharts@3.9.2` in apps/web deps, `ui/chart.tsx` (shadcn-scaffolded), `recharts-test-utils.tsx`
(ResponsiveContainer mock), `zorder-spike.test.tsx` (A1 proof), and a reconciled D3 stack-decision.
</objective>
