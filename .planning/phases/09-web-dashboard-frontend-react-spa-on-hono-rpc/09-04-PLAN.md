<objective>
Build the thin-slice plumbing that gates the app and surfaces auth state: the typed `hc<AppType>()`
Hono RPC client (type-only `AppType` import — never runtime), the Supabase Auth gate (Login screen +
`onAuthStateChange`), a 401 interceptor that clears the cache and returns to Login, and the
AUTH_EXPIRED bottom banner driven by the 30s `GET /api/status` poll (UI-02). After this plan the app
authenticates and reacts to auth/session changes; the Overview screen + deploy land in Plan 05.

Purpose: This is the security spine of the SPA. Client gating is defense-in-depth (Phase 8 is the
real server gate); the value here is a clean auth UX (no 9s hang on 401) and the UI-02 banner.
Output: rpc client, auth gate, Login, AUTH_EXPIRED banner, status hook, with component tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Railway API | Untrusted SPA sends Supabase JWT as Bearer; Phase 8 is the real authority |
| Supabase Auth → browser | Session/JWT delivered to the SPA, persisted in localStorage by supabase-js |
| user paste/login input → app state | Email/password typed into Login; no eval, JSX-escaped |

## STRIDE Threat Register (ASVS L1)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Spoofing | rpc.ts Bearer header over transport | mitigate | `VITE_API_BASE_URL` MUST be `https://`; token sent only in the Authorization header, never query string or log |
| T-09-02 | Information Disclosure | JWT in localStorage (XSS-reachable) | accept | Single-user internal tool; supabase anon key is public + JWT is short-lived and server-validated; React JSX escaping is the XSS control; never `dangerouslySetInnerHTML` |
| T-09-03 | Elevation of Privilege | client-side auth gate bypass | accept | Client gating is defense-in-depth only; Phase 8 server enforces 401 on every read endpoint — bypassing the SPA gate yields no data |
| T-09-04 | Information Disclosure | token logging | mitigate | No `console.*` of session/access_token; only `console.warn`/`console.error` permitted and never with token payloads |
| T-09-05 | Tampering | 401 not clearing stale session | mitigate | 401 interceptor clears query cache + signs out + returns to Login (Task 1 + Task 3); stale authed UI cannot persist |
| T-09-SC | Tampering | npm installs (react/supabase/tanstack/charts) | mitigate | RESEARCH Package Legitimacy Audit approved all packages (official orgs); installs done in Plan 03; no `[SLOP]` packages |
</threat_model>
