<objective>
Expose the GEX read endpoint + MCP tool over the shared contract (SC-1, D-03/MCP-02), export the
Hono `AppType` for typed RPC (SC-3, RPC-01), and gate the read endpoints behind Supabase Auth + CORS
for the Vercel origin (SC-4, AUTH-01). This is the driving-adapter + auth slice; it composes the
08-05 getGex use-case and depends on the 08-01 docs un-deferral.

Per CONTEXT D-02 the read endpoints (status, journal, brokerage, analytics, gex) move behind an
offline HS256 Supabase-Auth JWT verify; CORS allows the exact Vercel origin (never wildcard with
credentials); the existing `/api/jobs/*` bearerAuth group and `/mcp` stay unchanged. Per RPC-01 the
read routes are refactored into one chained sub-router so `hc<AppType>()` inference works, and
`AppType` is exported.
Output: two new env vars (config + .env.example), `gex.routes.ts` + `get_gex` MCP tool, and the
main.ts refactor (CORS-first → chained apiRouter → JWT read group → AppType export).
</objective>
