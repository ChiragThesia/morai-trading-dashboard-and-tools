<objective>
Build the `DataTable<T>` primitive — one generic, presentational table component in the
`components/system/` molecule layer that both the Analyzer's `CandidateTable` (Plan 02) and
the Overview's `PositionsTable` (Plan 03) will render through. This is the foundation of the
whole phase: the "one reusable table" the user asked for.

DataTable owns the converged chrome (CONTEXT "Chrome reference = CandidateTable"): the
`w-full border-collapse font-mono text-[11px] tabular-nums` table root, the
`sticky top-0 z-10 bg-panel` header, dense `px-2 py-1.5` cells, sort/`aria-sort` header
affordance, row hover, and per-row `data-testid`. It stays purely presentational — sort,
selection, and hover STATE remain caller-owned (PATTERNS.md "sort state stays in the CALLER").

Purpose: eliminate the two independently-styled hand-rolled tables by giving both a single
column-def primitive to render through, so they become indistinguishable per the user's
screenshot complaint.
Output: `DataTable.tsx` + red-first `DataTable.test.tsx` + barrel export.

This is the FIRST generic (`<T>`) component in the codebase (RESEARCH Pitfall 7) — there is
no local idiom to copy; author it test-first.
</objective>
