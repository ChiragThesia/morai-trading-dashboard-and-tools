<objective>
Migrate `CandidateTable` from a hand-rolled `<table>` into a thin wrapper that builds
`DataTableColumn<PickerCandidate>[]` + rows and renders `<DataTable>`. This is Migration
Manifest step 2 — CandidateTable already has the "best" chrome and the richest existing test
coverage (via Analyzer.test.tsx / AnalyzerMobile.test.tsx), so migrating it first proves the
DataTable API against a real shape while its own public interface stays frozen.

The public API (`candidates`, `pastedCandidates`, `selectedId`, `combinedIds`, `sort`,
`onSortChange`, `onSelect`, `onToggleCombine`, `onRemovePasted`, `wrapperClassName`,
`tableClassName`, `wrapperTestId`) and every module-level export (`cycleSort`,
`sortCandidates`, `DEFAULT_CANDIDATE_SORT`, `compactCalendarName`, `CandidateSortKey`,
`CandidateSortState`) MUST stay byte-stable — Analyzer.tsx and AnalyzerMobile.tsx import these
directly and must need ZERO changes.

Purpose: converge CandidateTable onto the shared primitive without touching its callers.
Output: rewritten CandidateTable.tsx internals; existing Analyzer/AnalyzerMobile suites green.
</objective>
