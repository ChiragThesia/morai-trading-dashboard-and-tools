<objective>
Close Gap 1 (Truth 5: "A tested BSM engine can invert IV and compute greeks for any stored
observation"). Three defects in the compute engine permanently NaN-stamp valid observations
or fabricate IV:

- CR-02: `computeBsmGreeks.ts` passes the job wall-clock `now` to `computeT`, so any same-day
  (0DTE) row observed before the settlement cutoff but computed after it gets T=0 →
  `invertIv` returns `err({kind:'expired'})` → permanent NaN stamp. Fix: compute T from
  `obs.time` per row.
- CR-03: `iv-inversion.ts` uses the American intrinsic `max(K-S,0)`/`max(S-K,0)` as the
  lower-bound guard. SPX/SPXW are European; the correct lower bound is the discounted no-arb
  bound `K·e^(-rT) - S·e^(-qT)` (put) / `S·e^(-qT) - K·e^(-rT)` (call). Deep-ITM European puts
  that legitimately trade below American intrinsic are wrongly rejected and NaN-stamped.
- WR-01: the bisection endpoint-clamp branch and the non-converged Newton path can return
  `ok(sigma)` for a sigma that does not reprice to the mark — fabricated IV stored as valid.
  Fix: a post-solve residual check; if `|bsmPrice(...) - mark|` exceeds tolerance, return err.

Purpose: Without these fixes the journal silently destroys or fabricates the exact short-DTE
data this phase exists to collect. This plan makes "any stored observation" true.
Output: Corrected `computeBsmGreeks.ts` (obs.time T) and `iv-inversion.ts` (European bound +
residual check), each driven by failing regression tests first (RED→GREEN).
</objective>
