<objective>
Close UAT Gap A: production `fetch-cboe-chain` inserts have failed every RTH slot
(`leg_observations` has 0 rows ever) because `persistObservations` issues ONE
`db.insert().values(allRows)`. A full SPX chain (≈12,556 rows × 14 columns) produces
175,784 bind parameters in a single INSERT — far past Postgres's 65,534-parameter
per-statement protocol limit. `upsertContracts` has the identical unbounded-batch shape
(8 columns × ≈12,556 rows ≈ 100,448 params), so it would fail the same way once contract
volume grows.

Fix: chunk both batched inserts at a fixed row cap (2,000 rows per INSERT → ≤28,000 params
for observations, ≤16,000 for contracts — comfortable margins), preserving
`onConflictDoNothing` idempotency per chunk. No schema or port-signature changes.

Purpose: restore the only write path that fills `leg_observations` — without it the entire
fetch → compute → status pipeline produces nothing downstream.
Output: chunked repo methods + a large-batch regression test that fails on the unchunked
code and passes after.
</objective>
