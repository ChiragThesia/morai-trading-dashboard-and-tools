<objective>
The FRED rate vertical slice: fetch the DGS3MO 3-month risk-free rate from FRED behind
`ForFetchingRate`, fall back to 4.5% when FRED is unreachable or no API key is configured
(D-02/D-13), filter FRED's `.` missing-value sentinel, and persist into `rate_observations`.
End-to-end (port → FRED adapter + in-memory twin → repo → fetchRate use-case). Implements
MKT-02.

Purpose: The BSM compute job (Plan 06) needs a stored risk-free rate `r`; this slice
supplies it with a guaranteed fallback so compute never blocks on FRED. Vertical slice per
MVP mode. Reuses the StorageError + repo + contract-harness patterns from Plan 04.
Output: ForFetchingRate port, FRED + in-memory rate adapters (contract-tested with msw),
rate-observations repo (testcontainers), fetchRate use-case.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Information Disclosure | FRED_API_KEY in logs | mitigate | D-13: never log config values; fallback warn message contains static text + field names only, never the key |
| T-02-12 | Denial of Service | FRED unreachable blocks compute | mitigate | 4.5% fallback returns ok on any network/parse failure so compute never blocks; tested with msw 500 + network error |
| T-02-13 | Tampering | malformed FRED payload | mitigate | observations Zod-parsed; '.' rows filtered; parse failure → fallback, never throw |
| T-02-14 | Tampering | SQL injection via rate date | mitigate | Drizzle parameterized upsert on date PK; no raw interpolation |
