<objective>
The final vertical slice: make the three use-cases run on a schedule and make their
outcomes visible. Wires fetch-cboe-chain, fetch-rates, and compute-bsm-greeks as scheduled
pg-boss jobs in the worker with double-layered RTH gating (D-06) and chain→compute enqueue
(D-07), and extends `lastJobRuns` end-to-end — contracts schema → core StatusPayload → both
the HTTP route and MCP tool (MCP-02) — to report per-job success/error from `pgboss.job`
(D-10). After this plan, a deployed worker collects data on cadence and `/api/status`
shows it. Covers the scheduling requirement (SPEC req 7) and operationalizes MKT-01/02/03 +
BSM-03.

Purpose: Without scheduling + visibility, the pipeline built in Plans 04-06 never runs in
production and failures are invisible. This is the slice that turns the use-cases into a
live collection loop. No manual trigger surface (D-08) — that is Phase 5.
Output: RTH window util, job-runs repo, extended lastJobRuns across contracts/core/both
adapters, three thin handlers, worker composition root scheduling all three jobs.
</objective>
