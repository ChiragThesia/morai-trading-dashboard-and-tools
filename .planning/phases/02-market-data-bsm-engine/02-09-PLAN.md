<objective>
Close Gap 2 (Truth 6: "Worker schedules three pg-boss jobs on boot and the pipeline runs
end-to-end"). The worker cannot boot on a fresh database:

- CR-01: pg-boss v12 requires `await boss.createQueue(name)` for every queue before
  `boss.schedule()`/`boss.send()` (FK constraint on the schedule table). `apps/worker/src/main.ts`
  calls `boss.schedule()` for all three jobs with no preceding `createQueue` — on a fresh deploy
  it throws "Queue <name> not found" and Railway restart-loops. Fix: three idempotent
  `boss.createQueue()` calls right after `boss.start()` and before the first schedule/work/send.
- WR-02: `apps/worker/src/handlers/fetch-cboe-chain.ts` does `void deps.boss.send('compute-bsm-greeks',
  ...)` with no `.catch()`. A failed enqueue (queue missing, transient DB error) becomes an
  unhandled rejection (process risk) or silently drops the D-07 chain trigger. Fix: attach a
  `.catch()` that logs via console.warn so the chain job's containment promise is actually honored.

Purpose: Without createQueue the entire pipeline never runs in production — every other Phase 2
deliverable is dead code on a real deploy. This plan makes the worker boot and the chain trigger
fail loud-but-contained.
Output: `apps/worker/src/main.ts` with three createQueue calls in the correct order;
`fetch-cboe-chain.ts` with a caught boss.send; a handler test asserting the enqueue failure is
contained.

Independence note: this plan touches only `apps/worker/**`. Sibling gap plan 02-08 (same wave)
touches only `packages/core/**` and explicitly preserves the makeComputeBsmGreeksUseCase factory
signature (it keeps the `now` dep), so the composition-root wiring here needs no reconciliation —
the two plans are fully parallelizable with zero file overlap.
</objective>
