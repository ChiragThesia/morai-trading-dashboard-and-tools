<objective>
The pure Black-Scholes-Merton pricing + greeks engine in the hexagon. European
calls/puts with continuous dividend yield q (D-01). Greeks stored in TOS display
convention (D-12). This is the numerical core that Plan 03 (IV inversion) and Plan 06
(compute-bsm-greeks use-case) build on. Implements BSM-02. Written fresh per D-05 — the
trade-advisor `bsm.ts` is a READ-ONLY reference (mine the A&S ncdf coefficients + formula
structure), never copied, because it is impure (module cache, hardcoded r/q, no Result).

Purpose: Phase 2's whole value is trusting our own computed greeks against the trader's
TOS screen (CONTEXT specifics). Numerical code requires fast-check property tests per
tdd.md; this plan is TDD because behavior is fully specified as input→output.
Output: `bsmPrice`, `bsmGreeks`, `bsmVega` exported from `packages/core` domain, all
validated against the 3 RESEARCH calibration fixtures at 1e-4.
</objective>
