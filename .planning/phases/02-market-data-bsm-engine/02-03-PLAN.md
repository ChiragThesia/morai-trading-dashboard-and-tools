<objective>
The IV-inversion solver: recover implied volatility from an option mark using
Newton-Raphson with analytic vega, falling back to bisection when Newton diverges or vega
is too flat (D-03). Returns `Result<number, IvError>` — degenerate marks yield typed
errors, never NaN, so the compute use-case (Plan 06) can stamp them per D-09. Implements
BSM-01.

Purpose: There is no canonical IV (RESEARCH iv-engine doc); we invert our own consistently.
Numerical solver behavior is fully testable as round-trip + monotonicity invariants, so
this is a TDD plan. The bisection fallback path MUST be property-covered (D-03).
Output: `invertIv` exported from `packages/core` domain, property-tested to 1e-6 round-trip.
</objective>
