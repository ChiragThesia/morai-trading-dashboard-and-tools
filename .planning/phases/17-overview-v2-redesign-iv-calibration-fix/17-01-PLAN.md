<objective>
Create `apps/web/src/lib/iv-calibration.ts` — the thin client-side caller that turns a live/REST
per-leg price into a calibrated implied vol via the already-shipped `invertIv` solver, or into a
tagged error. This is the OVW-02 wiring bridge: it replaces the flat `DEFAULT_IV` guess with a
real per-leg IV, and returns a `Result` that downstream code (scenario-engine, Overview) uses to
draw or to flag "IV n/a".

Per RESEARCH (HIGH confidence, direct repo inspection): `invertIv`
(`packages/core/src/journal/domain/iv-inversion.ts`) already exists, is production-hardened
(1000-run fast-check round-trip + monotonicity, Newton + 200-step bisection fallback, WR-01
post-solve residual guard), and is already the sole IV source for the live SSE stream and the
batch journal. **This plan does NOT build or modify a solver.** It builds one small caller and its
tests. Treat `invertIv`/`IvError` as frozen; import them from the `@morai/core` package root.

D-01: calibrate against the live mark (never raw bid/ask), return a tagged "did-not-converge"
result on failure — never the last iterate, never `DEFAULT_IV`.

This plan resolves RESEARCH Open Question 1 (do the two BSM engines agree?) with one smoke test,
without reconciling the engines (that is an out-of-scope architecture decision — Pitfall 5).

Purpose: One tagged, tested price→IV bridge so the payoff hero can show real per-leg vol and
honestly flag the legs it cannot price.
Output: `iv-calibration.ts` + `iv-calibration.test.ts`, TDD red→green, full suite green.
</objective>
