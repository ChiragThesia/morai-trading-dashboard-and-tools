# Phase 17 — UI Design Contract

> Visual and interaction contract for the Overview v2 "TOS dock" redesign + IV-calibration
> staleness/non-convergence surfacing. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**The layout is not open-ended.** `mockups/overview-v2.html` variant B ("TOS dock — positions
below graph ★ Selected") is the approved visual source of truth for OVW-01. This document
formalizes it into a checkable contract and specifies the STATEFUL behaviors (IV non-convergence,
staleness, row-highlight, scenario-strip bounds) the static mockup cannot pin down on its own.

Design system, tokens, and component primitives are LOCKED (established Phase 8/9, carried
through every screen since). This document does not re-derive them — it cites the existing
source of truth and states only what Phase 17 adds.

## Phase 17 State Contracts

The mockup fixes the layout skeleton; these are the stateful behaviors OVW-01/OVW-02 require
that the static HTML cannot show.

### A. IV non-convergence display (D-01 / D-02)

- **Source of truth:** `packages/core/src/journal/domain/iv-inversion.ts` → `invertIv(...)` returns
  `Result<number, IvError>` where `IvError.kind` is `"expired" | "below-intrinsic" | "above-bound"`.
  Treat **any** `err(...)` result uniformly for display — do not surface the specific `kind` to the
  user; it is an internal diagnostic detail, not a UI distinction.
- **Per-position row** (docked positions table): when a position's per-leg IV calibration returns
  `err`, render a `Badge` (variant `outline`, amber text/border — `text-amber`) reading **"IV n/a"**
  immediately after the position's name/label cell. Wrap it in the existing `Tooltip` pattern
  (see `LiveStatusBadge.tsx`) with body copy **"IV n/a — did not converge. @exp shown; excluded
  from T+0."**
- **Curve behavior for that leg:**
  - `@exp` (expiration) curve: computed and shown normally — intrinsic value at expiry needs no
    live IV, so this leg's contribution to the net-book `@exp` curve is unaffected.
  - `T+0` curve: this leg's contribution is **excluded** from the net-book T+0 sum. Never
    substitute `DEFAULT_IV`, never draw a fabricated per-leg T+0 curve, never silently keep the
    leg in the T+0 total with a guessed number.
- **Net-book self-flagging:** when one or more positions are excluded from the T+0 aggregate,
  render a small inline note in the payoff Panel (placed beside/under the existing chart legend
  row, matching the mockup's `.legend` position): **"T+0 excludes {n} position{s}: IV n/a"**, in
  `text-amber` at label size (10px).
- **Explicitly out of scope this phase:** no retry/recalibrate control, no page-level error toast,
  no blocking state — the rest of the dashboard renders normally around the flagged leg.

### B. Staleness surfacing — two-channel (D-03 / D-04)

Two independent badges, both following the SAME existing visual grammar already shipped in
`Market.tsx` (colored dot + label + timestamp + relative age) — do not invent a second staleness
language for the same concept.

- **GEX badge — reuse verbatim, no changes.** `Market.tsx` already implements this exactly:
  `gexTs = new Date(gex.computedAt)`, `GEX_FRESH_MS = 35 * 60 * 1000` (~35 min, already covers the
  ~30-min refresh cadence + margin — this constant is the resolution of D-03's "~45 min"
  recommendation; reuse the shipped constant rather than introducing a second unreconciled
  threshold), `relAge(ms)` helper, dot color `bg-up`/`bg-amber`. Overview's GEX rail imports and
  renders this exact badge — do not reimplement.
- **Live-mark badge — new for Phase 17.** Overview already has `liveTs` per row/net-total via
  `resolveLivePositionRow` (`Overview.tsx`), and a stream-*connection* badge (`LiveStatusBadge`,
  states `live/stale/reconnecting/poll`) — that badge answers "is the pipe connected," which is
  Phase 20's watchdog concern, **not** this phase's. D-03 needs an independent, simple **age**
  check on the mark timestamp itself:
  - Compute `markAgeMs = Date.now() - new Date(liveTs).getTime()` from the net-book total's
    `liveTs` (or the most recent per-row `liveTs` if the net total has none).
  - Render one badge near the payoff hero header: dot + **"mark as of {HH:MM}"** + relative age
    (reuse `relAge` from `Market.tsx` or an equivalent local helper — do not duplicate the format).
  - Tint amber (dot `bg-amber`, text `text-amber`) when `markAgeMs > 5 * 60 * 1000` (5 min) —
    covers both a stalled stream and being outside RTH. Tint `bg-up`/`text-up` otherwise.
  - **This threshold is independent of `LiveStatusBadge`'s connection state** — a `stale`/
    `reconnecting` connection and an age-stale mark can differ (e.g. reconnect succeeds but the
    last tick is still >5 min old at outside-RTH open).
- **Outside RTH / no live tick ever received:** freeze the payoff to the last known mark (or the
  REST-polled positions snapshot if no tick has ever arrived), keep the "mark as of {time}" badge
  amber, and **never blank the payoff chart** — it always renders something with a timestamp.
- **Placement:** both badges sit in the "Risk profile — combined book" Panel's header row,
  beside/replacing the mockup's static `<span class="badge-live">` — same position, same visual
  weight, now driven by real thresholds instead of a hardcoded "LIVE" label.

### C. Payoff hero scope & row-highlight interaction (D-05)

- The payoff hero is the **net book only** — one T+0 curve, one `@exp` curve, computed the same
  way `PayoffChart` already renders them today (no per-position curve toggle, no legend
  explosion). This matches OVW-01's "net book greeks" + docked positions table below it.
- **Row highlight (new interaction, not in the static mockup):** hovering OR clicking (selecting)
  a row in the docked positions table re-renders `PayoffChart` with that ONE position's T+0/`@exp`
  curves drawn at full emphasis, while the net-book curves dim — they are **never removed**.
  - Highlighted single-position curves: same stroke tokens as the net-book curves today (T+0
    violet `#a78bfa` @ 2.6px with glow filter; `@exp` gray-muted dashed @ 1.4px) so the visual
    grammar for "this is a T+0/@exp curve" stays consistent.
  - Dimmed net-book curves: reduce stroke-opacity to **0.3** (a chart-layer opacity change, not a
    DOM/row opacity change). Do **not** reuse the existing `opacity-40` row class from
    `PositionsTable`'s exclude-from-total checkbox — that is a different concept (excluded from
    the sum) and must stay visually distinct from "dimmed because another row is highlighted."
  - Hover reverts on mouseleave. Click/select persists until a different row is selected, or the
    same row is clicked again to clear (mirrors the existing `clearHovered` toggle pattern in
    `AdHocPicker.tsx`).
  - **No modal, no drill-down route, no separate chart instance** — same `PayoffChart` component,
    same `Panel`, in place. A dedicated single-position analyzer is an explicitly deferred idea
    (see 17-CONTEXT.md `<deferred>`), not part of this phase.

### D. Scenario-strip key-level set (D-06 / D-07)

- **Level set, left to right:** put wall → γ flip → spot → call wall → each position's short/long
  strikes (deduped against the four GEX levels and each other, sorted ascending by strike).
- **Cap:** total strip columns capped at **8** (the 4 GEX levels + up to 4 additional
  position-strike columns). If more than 4 unique position strikes remain after dedup, keep the 4
  closest to spot and silently drop the rest — no "…" overflow affordance needed at the current
  single-user, 2-calendar book scale; revisit the cap if the book grows.
- **Layout:** reuse the mockup's grid strip (`grid-template-columns: 70px repeat(N,1fr)`, three
  rows — level labels / T+0 values / `@exp` values). Translate the mockup's inline
  `background:var(--raise)` cell styling to `bg-raise` + the existing `signClass`/`text-up`/
  `text-down` sign-coloring convention already used in `Overview.tsx`.
- **`@exp` column header must name the expiry**, not read a bare "@exp": e.g. **"@ exp (Nov 20)"**.
  Source the date from the book's FRONT (nearest) expiry across all included positions — `
  scenario-engine` already models `frontDte`; convert to a calendar date and format it with the
  same short month/day convention already used for `gexAsOf` in `Market.tsx` (`toLocaleString`,
  `{ month: "short", day: "numeric" }`). For a multi-expiry book, this is the single earliest
  front expiry across positions (a book-level header, not per-position).

### E. Layout structure (OVW-01 — variant B "TOS dock")

Match `mockups/overview-v2.html` variant B exactly; deviate only where real data/existing working
code forces it, and record the deviation here.

1. **Pill header** (sticky, full width): SPX spot · net γ/1% + regime chip · γ flip · VIX · VVIX ·
   Fed funds (DFF) · 10y−2y · COT lev · book P&L. Reuse the `MetricChip`/pill visual language
   already shipped (`Market.tsx` `MetricChip`, mockup `.pill`) — no new pill component.
2. **Two-column body:**
   - Left column (flex, fills remaining width): payoff hero Panel ("Risk profile — combined book")
     stacked above the docked positions table Panel ("Positions").
   - Right column (fixed **320px**, matches mockup `cockpit-b`): GEX rail, stacked Panels in this
     order — Dealer γ profile (compact `GammaProfile`) → GEX by strike (compact `GexBars`, mode
     locked to `gex`) → Key levels → Net book greeks.
3. **Below the two-column body**, unchanged from the current `Overview.tsx` (Pitfall 6 — do not
   rewrite what already works):
   - "Positioning & macro detail" row: `CotCard` + `MacroCard` side by side.
   - "Book & system" row: `BookSummary` + `SystemHealth` side by side.
4. **What actually changes vs. today's `Overview.tsx`:**
   - Current Section 1 ("Open positions · greeks" — a plain table, no chart) is **replaced** by
     the new payoff-hero + docked-table pair.
   - Current Section 2's full-width `<Market />` embed is **replaced** by the compact right-rail
     GEX stack (reusing `GammaProfile`'s and `GexBars`' existing `compact`/sizing props) — the
     full, non-compact Market screen stays reachable unmodified via its own nav tab (not deleted,
     not degraded).
   - `CotCard`, `MacroCard`, `BookSummary`, `SystemHealth` are reused **unmodified** — only their
     position in the page shifts.

## Regression Guardrails (Pitfall 6 — re-run against the new screen)

- SPX OI=0 → SPY proxy scaling still displays correctly wherever GEX/OI numbers appear in the new
  right-rail placement.
- GEX put-sign convention (negative gamma for puts) unchanged in the compact `GammaProfile`/
  `GexBars` placement.
- CBOE timestamp-is-UTC handling unchanged — the new "mark as of" / "GEX as of" badges must both
  convert to local time correctly, not display a raw UTC string.
- The full (non-compact) Market screen remains reachable via its nav tab with no regression —
  this redesign narrows Overview's GEX rail, it does not remove or degrade the dedicated screen.
