<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| calibration result → payoff hero | a fabricated/flat IV must never render on the hero as if it were the live-mark-implied vol |
| snapshot/mark timestamps → staleness UI | stale data must self-declare; a raw UTC string or a missing "as of" would let the user read stale as live |
| live-stream ticks → single consumer | multiple `useLiveStream()` calls would double-subscribe and desync book state |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-17-05 | Tampering | `DEFAULT_IV` silently rendered on the payoff hero as if calibrated | high | mitigate | Hero path prices exclusively via `resolveLegIv`/tick `bsmIv` with tagged status; `DEFAULT_IV` kept only on the recorded-deferred `BookSummary` path, never the hero (Task 2, verified by grep + test) |
| T-17-06 | Information disclosure | stale GEX / stale live-mark shown as if live (no timestamp, or raw UTC) | medium | mitigate | Two-channel staleness — always-visible "as of" timestamp + amber threshold; CBOE-UTC → local conversion (D-03/D-04, Task 2) |
| T-17-09 | Tampering | cold-start "no tick" misread as non-convergence → false "IV n/a" on healthy legs | medium | mitigate | `resolveLegIv` returns distinct `no-price` vs non-convergence; only non-convergence badges "IV n/a" (Pitfall 2, Task 2 test) |
| T-17-10 | Denial of Service | duplicate `useLiveStream()` consumers desync/oversubscribe the SSE stream | low | mitigate | Single-`useLiveStream()` invariant preserved and grep-verified (Task 1) |
</threat_model>
