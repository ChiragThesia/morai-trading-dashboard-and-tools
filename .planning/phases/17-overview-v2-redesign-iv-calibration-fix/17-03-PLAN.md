<objective>
Extend the existing `PayoffChart.tsx` (do NOT create a second chart component or a modal) with the
row-highlight dual-curve mode (D-05) and the net-book non-convergence self-flag note (D-02), both
specified by 17-UI-SPEC.md State Contracts A & C.

Two visual behaviors, both driven by new props (Plan 04 supplies the values):

1. **Highlight (D-05):** when `highlightedPositionId` is set, the net-book T+0/@exp curves dim to
   `stroke-opacity: 0.3` (a chart-layer change) and the single highlighted position's T+0/@exp
   curves render at full emphasis using the SAME stroke tokens as today's net-book curves (T+0
   violet `#a78bfa` @ 2.6px with glow; `@exp` gray-muted dashed @ 1.4px). The net-book curves are
   **dimmed, never removed**. This 0.3 chart-opacity change is deliberately distinct from
   `PositionsTable`'s `opacity-40` exclude-from-total row class — different concept, must stay
   visually distinct (UI-SPEC explicit).

2. **T+0 exclusion note (D-02):** when the net book excludes ≥1 position from the T+0 aggregate,
   render a small amber note near the legend row reading "T+0 excludes {n} position(s): IV n/a"
   (singular "position" when n=1). No note when the count is 0.

This plan owns only `PayoffChart.tsx` + its (new) test file — no overlap with Plans 01/02, so it
runs in parallel in Wave 1. The curve DATA for a highlighted single position is computed by Plan 04
from scenario-engine; this plan only accepts it as props and renders it.

Purpose: The payoff hero can spotlight one position's contribution and honestly label the legs it
dropped, without a modal or a second chart.
Output: extended `PayoffChart.tsx` + new `PayoffChart.test.tsx`, TDD red→green, suite green.
</objective>
