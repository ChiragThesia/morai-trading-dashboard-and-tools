<objective>
WATCH-01 client side: wire the currently-ignored `ping` handler, add the elapsed-time interval that
derives LIVE/QUIET/CONNECTING/STALLED, and restyle the badge to the 3-state honest indicator with a
force-reconnect action. This is the plan that closes the Phase-12 "badge lies LIVE" debt.

Purpose: Silence is the signal — the stall threshold and cold-start grace are elapsed-time
conditions that must fire even when no event arrives, so the status must come from a shared timer
reading `deriveStreamStatus`, not from transport event handlers (Pitfall 1). The badge must read as
a genuine alarm when STALLED (D-20), and offer an immediate, backoff-cancelling reconnect (D-17).
Output: honest 3-state badge, server-authoritative, with working force-reconnect.
</objective>
