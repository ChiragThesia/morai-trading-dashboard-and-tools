<objective>
RULE-01 storage: the orthogonal annotations table that survives journal rebuilds. Docs-first
(stack-decisions.md records the deliberate no-FK design), then migration 0017 + Drizzle schema +
Postgres repo + memory twin + contract-parity test, plus the rebuild-survival regression guard that
proves D-09.

Purpose: rebuildJournal is delete-then-reinsert, so any column on calendar_events is wiped every
rebuild (the latent entryThesis data-loss bug). A real FK would either CASCADE-wipe annotations or
RESTRICT-break the rebuild — the correct behavior is explicitly the OPPOSITE of referential
integrity (Pitfall 3). The regression test locks that in.
Output: a live-ready, empty, no-FK annotations table with proven memory/Postgres parity and a
rebuild-survival guard.
</objective>
