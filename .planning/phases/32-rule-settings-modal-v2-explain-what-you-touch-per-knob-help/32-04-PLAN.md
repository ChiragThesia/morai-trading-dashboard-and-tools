(B4/B7/B8). The combined use-case is the single seam both adapters call so HTTP and MCP can never
drift — the same "hand-copies rot" lesson CONTEXT calls out, applied to the adapter surface.

route only branches picker/exits, mirroring the existing PUT's group-branching shape.

`settings.routes.ts` (+ tests); the MCP tool + registration + `main.ts` composition.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API (HTTP + MCP) | staged overrides cross the authenticated boundary into a read-only compute |
| use-case→storage | bounded reads only (stored snapshot re-score, open-position re-eval); never a write |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-32-05 | Input Validation | previewRuleOverridesRequest | high | mitigate | zValidator on the reused `ruleOverrides` schema (`.strict()`, weight-sum-100, hysteresis) rejects malformed bodies with 400 before the handler runs |
