<objective>
Expose parity-implied carry on the GEX snapshot end-to-end: a new additive, nullable `impliedCarry`
field carrying, per expiry, the resolved risk-free rate (interpolated from the live FRED curve the
system already ingests) and the dividend yield implied by put-call parity from our own stored chain.
This is the production-and-delivery half of CONTEXT item 2; the browser consumes it in plan 34-05.

Purpose: the flat DEFAULT_DIV (0.013) and DEFAULT_RATE (0.045) guesses are a second independent source
of BE-point error. RESEARCH decomposes the fix into fixing r from FRED and solving the single unknown q
from parity (D-06 / Pattern 2). Computing this once per GEX cycle server-side, over the ATM-bracket
marks already in the cohort, and emitting BOTH resolved numbers per expiry lets the browser re-price
each leg with the exact (r, q) the parity solve used — killing both r and q error jointly with no raw
marks leaving the server and no client/server rate drift.

Output: `impliedCarry` on `gexSnapshotEntry` + `GexSnapshotRow`; migration 0023 (nullable jsonb column);
persist/read mapping; the per-expiry compute in `computeGexSnapshot` with a FRED-rate dep; and the
GET /api/analytics/gex + get_gex MCP passthrough (rule 9).
</objective>
