<objective>
Deliver the one genuinely new pure function this phase needs: `settlementTimestamp(root, expiry)`,
which converts an OCC-parsed contract's root + expiry-date into the exact wall-clock settlement
instant (09:30 ET for AM-settled standard SPX, 16:00 ET for everything else), DST-safe via native
`Intl`. This is the numerator feeding fractional time-to-expiry in plan 34-02 — the highest-impact
TOS-parity lever (CONTEXT item 1).

Purpose: `pair-calendars.ts`'s `dte()` rounds to whole calendar days; at night a "98d" leg is really
~97.6d and the book carries ~$32/day theta, so whole-day rounding is worth ~10-30 BE points against
TOS. TOS prices exact time to settlement. This helper supplies the exact settlement instant with zero
new data (root + date already come from `parseOccSymbol`) and zero new dependency (native `Intl`,
mirroring the existing `rth-window.ts` ET-reading technique).

Output: `settlementTimestamp` exported from `@morai/shared`, with an example + DST-boundary oracle test
and the A1 anchor flagged as a single named constant.
</objective>
