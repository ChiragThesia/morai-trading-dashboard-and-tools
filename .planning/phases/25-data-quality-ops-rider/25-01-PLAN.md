<objective>
OPS-01: stop snapshot-calendars from writing silent gap rows (spot=0 / net_mark=0 / front_iv=NaN)
when an open calendar has no fresh-enough leg data. Root-cause fix in the write path — a cycle
with no fresh cohort for a calendar SKIPS that calendar's row (next cycle self-heals) instead of
persisting zeros or silently serving stale marks. Never gap-fill historical rows.

Purpose: The Exit Advisor (Phase 26) and Backtest (Phase 27) read calendar_snapshots directly. A
zero/NaN gap row becomes a confident wrong verdict once an advisor consumes it. This closes the
Jul-06 corruption mechanism before those features inherit it.

Output: `LegSnapshot.time` threaded through both read paths + both twins; a documented, tested
freshness gate (`SNAPSHOT_LEG_STALENESS_TOLERANCE_MS`) in the snapshotCalendars use-case; corrected
jobs.md behavior row. No new tables, no contract-schema changes.
</objective>
